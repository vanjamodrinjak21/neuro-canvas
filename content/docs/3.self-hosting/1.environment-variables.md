# Environment Variables

NeuroCanvas uses environment variables for all sensitive configuration. In Docker deployments, set these in your `.env` file or pass them directly in `docker-compose.yml`. For local development, create a `.env` file in the project root.

## Required Variables

These variables must be set for the application to start and function correctly.

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string. Must include the database name. | `postgresql://user:password@localhost:5432/neurocanvas` |
| `AUTH_SECRET` | Secret key used by NextAuth.js for signing tokens, encrypting sessions, and deriving encryption keys. **Must be at least 32 characters.** | `a1b2c3d4e5f6...` (64 hex characters) |
| `AUTH_ORIGIN` | The canonical base URL of your deployment. Used for OAuth callback URLs and CORS. Must not include a trailing slash. | `https://neuro-canvas.com` |

### Generating AUTH_SECRET

The `AUTH_SECRET` is the most critical security variable. It is used for:
- Signing and verifying JWT session tokens
- Deriving the master encryption key for the credential vault (AES-256-GCM via scrypt)
- Deriving per-user key encryption keys (HMAC-SHA256)

Generate a cryptographically secure secret:

```bash
# Using openssl (recommended)
openssl rand -hex 32

# Using Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Using Python
python3 -c "import secrets; print(secrets.token_hex(32))"
```

::callout{type="danger"}
Never reuse `AUTH_SECRET` across environments. Changing this value invalidates all existing sessions and makes previously encrypted credential vault entries unrecoverable. Back up the value securely.
::

## OAuth Provider Variables

At least one OAuth provider or email credentials should be configured for users to sign in. All OAuth variables are optional individually, but the app is most useful with at least one provider enabled.

### Google OAuth

| Variable | Description | Example |
|----------|-------------|---------|
| `GOOGLE_CLIENT_ID` | OAuth 2.0 Client ID from Google Cloud Console | `123456789.apps.googleusercontent.com` |
| `GOOGLE_CLIENT_SECRET` | OAuth 2.0 Client Secret | `GOCSPX-abc123...` |

**Setup instructions:**

1. Go to the [Google Cloud Console](https://console.cloud.google.com/apis/credentials).
2. Create a new project or select an existing one.
3. Navigate to **APIs & Services > Credentials**.
4. Click **Create Credentials > OAuth client ID**.
5. Select **Web application** as the application type.
6. Set the **Authorized redirect URI** to: `https://your-domain.com/api/auth/callback/google`
7. Copy the generated Client ID and Client Secret.

::callout{type="info"}
For local development, add `http://localhost:3000/api/auth/callback/google` as an additional authorized redirect URI.
::

### GitHub OAuth

| Variable | Description | Example |
|----------|-------------|---------|
| `GITHUB_CLIENT_ID` | OAuth App Client ID from GitHub Developer Settings | `Iv1.abc123def456` |
| `GITHUB_CLIENT_SECRET` | OAuth App Client Secret | `abcdef1234567890...` |

**Setup instructions:**

1. Go to [GitHub Developer Settings](https://github.com/settings/developers).
2. Click **New OAuth App**.
3. Fill in the application details:
   - **Application name:** NeuroCanvas (or your preferred name)
   - **Homepage URL:** `https://your-domain.com`
   - **Authorization callback URL:** `https://your-domain.com/api/auth/callback/github`
4. Click **Register application**.
5. Copy the Client ID. Click **Generate a new client secret** and copy it.

::callout{type="warning"}
GitHub OAuth Apps and GitHub Apps are different. NeuroCanvas uses an **OAuth App**, not a GitHub App.
::

## Email Variables

Email is used for credentials-based authentication (sign up with email/password), verification emails, and password reset flows.

| Variable | Description | Default |
|----------|-------------|---------|
| `RESEND_API_KEY` | API key from [Resend](https://resend.com) for transactional email delivery | *(none)* |
| `EMAIL_FROM` | The sender address for outgoing emails. Must be a verified domain in your email provider. | `NeuroCanvas <noreply@neurocanvas.app>` |

**Resend setup:**

1. Sign up at [resend.com](https://resend.com).
2. Verify your sending domain under **Domains**.
3. Create an API key under **API Keys**.
4. Set `EMAIL_FROM` to an address on your verified domain.

If `RESEND_API_KEY` is not set, email-based authentication (sign up, email verification, password reset) will not function. OAuth providers will still work.

## Caching Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `REDIS_URL` | Redis connection string. Used for caching AI responses, embedding vectors, sync state, distributed locks, and rate limiting. | `redis://localhost:6379` |

Redis is **optional but strongly recommended** for production deployments. Without Redis:
- AI response caching is disabled (every request hits the LLM provider)
- Sync state must be read from PostgreSQL on every request
- Rate limiting for the sync API is disabled
- Distributed locks for concurrent map edits are unavailable

### Redis with Authentication

If your Redis instance requires a password:

```
REDIS_URL=redis://:your-redis-password@localhost:6379
```

For Redis with TLS (e.g., AWS ElastiCache, Upstash):

```
REDIS_URL=rediss://:your-redis-password@your-host:6380
```

Note the `rediss://` protocol (double s) for TLS connections.

## Runtime Variables

These are set automatically by the Docker container or Node.js runtime. You generally do not need to configure them manually.

| Variable | Description | Default |
|----------|-------------|---------|
| `NODE_ENV` | Node.js environment. Set to `production` for deployments. | `development` |
| `HOST` | Host to bind the HTTP server to. | `0.0.0.0` |
| `PORT` | Port to bind the HTTP server to. | `3000` |

## Full Reference Table

| Variable | Required | Server-only | Default |
|----------|----------|-------------|---------|
| `DATABASE_URL` | Yes | Yes | *(none)* |
| `AUTH_SECRET` | Yes | Yes | *(none)* |
| `AUTH_ORIGIN` | Yes | No (public) | `http://localhost:3000` |
| `GOOGLE_CLIENT_ID` | No | Yes | *(none)* |
| `GOOGLE_CLIENT_SECRET` | No | Yes | *(none)* |
| `GITHUB_CLIENT_ID` | No | Yes | *(none)* |
| `GITHUB_CLIENT_SECRET` | No | Yes | *(none)* |
| `RESEND_API_KEY` | No | Yes | *(none)* |
| `EMAIL_FROM` | No | Yes | `NeuroCanvas <noreply@neurocanvas.app>` |
| `REDIS_URL` | No | Yes | `redis://localhost:6379` |
| `NODE_ENV` | No | Yes | `development` |
| `HOST` | No | Yes | `0.0.0.0` |
| `PORT` | No | Yes | `3000` |

"Server-only" means the variable is never exposed to the client-side JavaScript bundle. Only `AUTH_ORIGIN` is available on the client (as `runtimeConfig.public.authOrigin`).

## Example .env File

```bash
# ─── Database ────────────────────────────────────────────
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/neurocanvas"

# ─── Authentication ──────────────────────────────────────
AUTH_SECRET="generate-this-with-openssl-rand-hex-32"
AUTH_ORIGIN="https://your-domain.com"

# ─── OAuth Providers ─────────────────────────────────────
GOOGLE_CLIENT_ID="your-google-client-id.apps.googleusercontent.com"
GOOGLE_CLIENT_SECRET="GOCSPX-your-google-secret"
GITHUB_CLIENT_ID="Iv1.your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"

# ─── Email ───────────────────────────────────────────────
RESEND_API_KEY="re_your_resend_api_key"
EMAIL_FROM="NeuroCanvas <noreply@your-domain.com>"

# ─── Caching ─────────────────────────────────────────────
REDIS_URL="redis://localhost:6379"
```
