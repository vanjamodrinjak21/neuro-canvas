# Export and Import

NeuroCanvas supports multiple export formats so you can back up your work, share it with others, or integrate it into other tools.

## Export formats

### JSON (full data export)

JSON export produces a complete, machine-readable representation of your map -- every node, edge, position, style, description, and piece of metadata.

**When to use it:**
- Backing up a map for safekeeping.
- Transferring a map between devices (e.g., from the desktop app to the web).
- Sharing a map with another NeuroCanvas user who wants to edit it.
- Version control (store JSON files in Git to track map evolution over time).

**How to export:**
1. Open the map you want to export.
2. Open the overflow menu (three dots icon) in the top bar.
3. Select **Export as JSON**.
4. A `.json` file downloads with a filename based on your map title.

**What the file contains:**
- Map metadata (title, description, creation date, last modified date).
- All nodes with their positions, sizes, styles, content, descriptions, and metadata.
- All edges with their source/target references, labels, and styles.
- Root node identifier.
- Semantic data (embeddings, similarity cache) if available.

### PNG (visual snapshot)

PNG export renders the current canvas view as a raster image.

**When to use it:**
- Embedding a map in a presentation, report, or document.
- Sharing a quick visual with someone who does not use NeuroCanvas.
- Creating a thumbnail or preview image.

**How to export:**
1. Arrange the canvas view to show what you want captured. Use `F` (fit to screen) to show the entire map.
2. Open the overflow menu in the top bar.
3. Select **Export as PNG**.
4. A `.png` file downloads.

::callout{type="info"}
The PNG captures the canvas at its current zoom level and viewport. For the best result, fit all nodes to screen before exporting so nothing is cut off.
::

### Markdown (text outline)

Markdown export converts your map's hierarchical structure into an indented text outline.

**When to use it:**
- Pasting map content into a document, email, or note-taking app.
- Creating a text-based summary of a visual map.
- Feeding map structure into other tools that accept Markdown.

**How to export:**
1. Open the map.
2. Open the overflow menu in the top bar.
3. Select **Export as Markdown**.
4. A `.md` file downloads.

**What the output looks like:**
The root node becomes an H1 heading. Tier-1 nodes become list items. Deeper nodes are indented further. Node descriptions, when present, are included below each item.

```markdown
# Machine Learning Fundamentals

- Supervised Learning
  - Classification
  - Regression
  - Decision Trees
- Unsupervised Learning
  - Clustering
  - Dimensionality Reduction
- Reinforcement Learning
  - Q-Learning
  - Policy Gradient Methods
```

### Share as text (mobile)

On mobile devices with share sheet support:

1. Open the map.
2. Tap the share button.
3. Choose **Share as text**.
4. The system share sheet opens with the Markdown outline ready to send to any app.

On desktop browsers, this copies the Markdown to your clipboard instead.

## Import

### Importing from JSON

To restore a map from a JSON export:

1. Go to the **Dashboard**.
2. Click **Import**.
3. Select a `.json` file from your filesystem.
4. NeuroCanvas parses the file and creates a new map with all the original data.
5. The imported map appears in your map list and can be opened normally.

**What gets imported:**
- All nodes, edges, and their properties.
- Node positions and styles (the layout is preserved exactly).
- Descriptions and metadata.
- The root node designation.

**What does not carry over:**
- Sync state (the imported map is treated as a new local map).
- Computed data like semantic embeddings (these are regenerated automatically).

### Handling import errors

If the JSON file is malformed or incompatible:

- NeuroCanvas validates the file structure before importing.
- If required fields are missing (e.g., nodes array, map title), the import fails with an error message.
- Partially valid files are not imported -- it is all or nothing to prevent corrupted maps.

## Backup and restore workflow

For regular backups:

1. **Export as JSON** periodically (weekly, or after major editing sessions).
2. Store the JSON files in a safe location -- cloud storage, a Git repository, or an external drive.
3. To restore, use the import flow described above.

### Cross-platform transfer

To move a map from the desktop app to the web (or vice versa):

1. On the source platform, export as JSON.
2. Transfer the file (email it to yourself, use cloud storage, or a USB drive).
3. On the target platform, import the JSON file.

This is the recommended workflow for desktop users since the Tauri app does not sync with the server.

### Version history

NeuroCanvas does not currently maintain built-in version history. To track how a map evolves over time:

1. Export as JSON at meaningful milestones.
2. Name the files with dates or version numbers (e.g., `thesis-map-2024-03-15.json`).
3. Optionally, store them in a Git repository to get diffs and a full timeline.
