# AI Features

NeuroCanvas integrates AI throughout the mind mapping workflow. All AI features use a BYOK (Bring Your Own Key) model -- you connect your own API keys, and processing happens through your chosen provider.

## Setting up AI

Before using AI features, configure at least one provider:

1. Go to **Settings** from the sidebar.
2. Open the **AI Providers** section.
3. Click **Add Provider** and select a provider type:
   - **OpenAI** -- GPT-4.1, GPT-4.1 Mini, GPT-4.1 Nano, o4-mini
   - **Anthropic** -- Claude models
   - **Ollama** -- Local models running on your machine (no API key needed)
   - **OpenRouter** -- Access many models through one API key
   - **Custom** -- Any OpenAI-compatible endpoint
4. Enter your API key (encrypted at rest) and select a model.
5. Click **Test Connection** to verify.
6. Set the provider as default if you want it used for all AI operations.

::callout{type="info"}
On the Tauri desktop app, local embedding inference runs via ONNX models bundled with the application. You still need an API key for generative features like map creation and node expansion.
::

## AI map generation

Generate an entire mind map from a text description.

### How it works

1. Open the **Generate Map** dialog from the dashboard (AI Quick Start) or from the overflow menu inside a map.
2. Enter a topic description. Examples:
   - "The history of the internet from ARPANET to modern day"
   - "Key concepts in behavioral economics"
   - "React server components architecture and data flow"
3. Configure options:

| Option | Choices | Effect |
|--------|---------|--------|
| **Depth** | Quick / Balanced / Comprehensive | ~15 / ~25 / ~40+ nodes |
| **Style** | Concise / Detailed / Academic | Amount of text per node |
| **Domain** | Free text | Focuses output on a specific field |

4. Click **Generate**. The AI constructs a hierarchical map with:
   - A root node representing the central topic.
   - Tier-1 branches for major subtopics.
   - Tier-2+ branches for supporting details, examples, and sub-categories.
   - Edges connecting related nodes with labeled relationships.
   - Descriptions for each node explaining the concept.

The generated map opens on the canvas, fully editable. Rearrange nodes, delete branches you don't need, or expand further with Smart Expand.

## Smart node expansion

Select any node and let the AI suggest contextually relevant child nodes.

### Basic expansion

1. Select a node on the canvas.
2. Press `Cmd/Ctrl + E`, or click **Smart Expand** in the AI Suggestions panel.
3. The AI analyzes:
   - The selected node's content and description.
   - Its parent node and siblings (to avoid redundancy).
   - The broader map structure and domain.
4. Suggestion cards appear in the **AI Suggestions** sidebar panel.
5. Each card shows:
   - **Title**: A concise label for the suggested idea.
   - **Description**: A sentence or two explaining the concept.
   - **Category**: What type of idea it is (e.g., "definition", "example", "technique").
   - **Relationship**: How it connects to the parent (e.g., "subtopic of", "example of", "contrasts with").
6. Click a suggestion card to add it to the map as a child of the selected node.

### Deep expansion

For multi-level expansion:

1. Select a node.
2. In the AI Suggestions panel, click **Deep Expand**.
3. The AI generates a multi-level hierarchy -- not just direct children, but grandchildren and great-grandchildren.
4. The entire sub-tree is added to the map at once, with edges and layout calculated automatically.

### Regeneration

Click **Generate more** in the AI Suggestions panel to get a fresh set of suggestions for the same node. The AI produces different results each time, so you can sample multiple perspectives.

## AI-powered descriptions

The AI can write descriptions for your nodes based on their title and map context.

- Select a node and open the properties panel.
- In the description section, click **Generate Description**.
- The AI produces:
  - A **summary**: 1-2 sentences explaining the concept.
  - **Details**: An extended description with more context.
  - **Keywords**: Tags useful for search and categorization.

Descriptions are stored as node metadata and used by the AI when expanding that node later, improving the quality of subsequent suggestions.

## Semantic field visualization

The semantic field is a visual layer that reveals meaning-based relationships between nodes.

### Enabling it

- Press `S` to toggle the semantic field layer on and off.
- Or use the toggle in the bottom toolbar.

### What you see

When enabled, the canvas displays:

- **Field lines**: Curved, animated lines connecting nodes that are semantically similar. Line thickness indicates similarity strength -- thicker lines mean stronger relationships.
- **Particles**: Small dots that flow along field lines. Speed is proportional to similarity. High-similarity connections have faster-moving particles.
- **Cluster glow**: Regions of the canvas where multiple related nodes are concentrated light up with a subtle ambient glow.
- **Resonance pulses**: When you select a node, a pulse radiates outward highlighting its semantic neighbors.

### How similarity is computed

NeuroCanvas generates **embedding vectors** for each node's content:

- **Web**: Uses transformers.js to run a lightweight embedding model in the browser.
- **Desktop (Tauri)**: Uses ONNX runtime for local inference with higher-quality models.

Cosine similarity between embeddings determines relationship strength. Only pairs exceeding the **similarity threshold** (configurable in settings, default 0.7) are visualized.

### Configuration

In **Settings > Preferences**:

- **Semantic field enabled**: Master toggle.
- **Similarity threshold**: Minimum cosine similarity for a field line to appear (0.0 to 1.0). Lower values show more connections; higher values show only the strongest.
- **Embedding model** (Desktop only): Choose between quantized (faster) and full (more accurate) ONNX models.

## AI Suggestions panel

The AI Suggestions panel lives in the right sidebar and acts as your AI collaborator.

### What it shows

- **Suggestion cards** from the last Smart Expand or Deep Expand operation.
- **Loading skeleton** while the AI is generating.
- **Empty state** with guidance when no node is selected or no suggestions have been generated.

### Actions

- Click a suggestion card to add it as a node connected to the selected parent.
- **Generate more**: Request another batch of suggestions for the same node.
- **Deep expand**: Request a multi-level hierarchy expansion.
- **Accept suggestion** (`Cmd/Ctrl + Enter`): Add the top suggestion automatically.

## Configuration options

### Personas

Personas let you customize AI behavior for different tasks:

- **System prompt**: Instructions that guide the AI's output style and focus.
- **Temperature**: Controls randomness. Lower (0.1-0.3) for factual, structured output. Higher (0.7-1.0) for creative, exploratory output.
- **Max tokens**: Maximum length of AI responses.
- **Top P, Frequency Penalty, Presence Penalty**: Fine-tuning parameters for advanced users.

Create multiple personas and switch between them depending on your task. For example:

- "Research Assistant" -- low temperature, academic style, thorough.
- "Brainstorm Partner" -- high temperature, creative, lots of varied suggestions.
- "Technical Writer" -- medium temperature, precise, well-structured.

### Preferences

In **Settings > Preferences**, toggle AI behaviors:

- **Auto-suggest connections**: AI automatically proposes edges between related nodes.
- **Semantic field enabled**: Toggle the visual similarity layer.
- **Streaming**: Enable SSE streaming for AI responses (see results as they generate).
- **LLM Insights**: Use the configured provider for deeper analytical insights.
