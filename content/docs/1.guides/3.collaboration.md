# Collaboration and Sharing

NeuroCanvas is primarily a personal thinking tool, but it provides several ways to share your work and keep it synchronized across your devices.

## Cross-device sync

When you are signed in on the web, your maps sync automatically.

### How sync works

1. Every change you make (adding, editing, or deleting nodes and edges) is saved to **IndexedDB** locally for instant persistence.
2. Changes are then **pushed to the server** with a debounced delay (a few seconds after your last edit) to avoid flooding the network on rapid changes.
3. On other devices signed into the same account, changes are **pulled via Server-Sent Events (SSE)**. Maps update in near real-time without manual refresh.
4. A **sync status indicator** in the UI shows the current state: idle, syncing, offline, conflict, or error.

### Offline support

If you lose network connectivity:

- All changes continue to save locally.
- The sync status switches to **offline**.
- When connectivity returns, pending changes are pushed automatically.
- If the same map was edited on another device while offline, the conflict resolution flow activates.

### Conflict resolution

When the same map has been edited on two devices simultaneously:

1. NeuroCanvas detects the conflict by comparing checksums and sync versions.
2. A **conflict dialog** shows both versions: local and remote, with node counts and timestamps.
3. You choose which version to keep, or merge manually.

### Desktop app

On the Tauri desktop app, maps are stored locally only. Sync is disabled because authentication is bypassed. To transfer maps between desktop and web, use the JSON export/import flow.

## Sharing maps

### Export for sharing

The most reliable way to share a map with someone else is to export it:

- **JSON export**: Produces a complete, re-importable file. The recipient can import it into their own NeuroCanvas account and get the full map with all nodes, edges, positions, and metadata. Best for sharing with other NeuroCanvas users.
- **PNG export**: Produces a visual snapshot of the canvas. Best for embedding in presentations, documents, or messages where the recipient does not need to edit the map.
- **Markdown export**: Produces a hierarchical text outline of the map. Best for pasting into documents, emails, or note-taking apps.

See the [Export & Import](/docs/guides/export-import) guide for step-by-step instructions.

### Share as text (mobile)

On mobile devices, NeuroCanvas can use the **native share sheet** to send a Markdown outline of your map to any app that accepts text -- messaging apps, email, notes, or cloud storage.

1. Open the map you want to share.
2. Tap the share/export button in the toolbar.
3. Choose "Share as text."
4. The system share sheet opens with the map rendered as a Markdown outline.

On desktop browsers, the same action copies the Markdown to your clipboard.

### Link sharing

Generate a shareable URL for any map so others can view it without an account:

1. Open the map you want to share.
2. Click the **Share** button in the toolbar (or use the overflow menu).
3. NeuroCanvas generates a unique share link.
4. Copy the link and send it to anyone. They can view the map in read-only mode without signing in.

Share links can optionally have an expiration date. The map owner can revoke a share link at any time.

## Real-time multi-user editing

::callout{type="warning"}
Real-time collaborative editing -- where multiple people work on the same map simultaneously with live cursors -- is planned but not yet implemented. The current sync system handles sequential edits from the same user across devices, not concurrent multi-user sessions.
::

When collaborative editing arrives, it will include:

- Live presence indicators showing who is currently viewing the map.
- Real-time cursor positions for each collaborator.
- Per-node locking to prevent conflicting edits on the same node.
- Change attribution so you can see who modified what.

## Best practices for team workflows

Even without real-time collaboration, you can work effectively in teams:

1. **One map owner, async feedback.** One person owns the map and shares PNG or Markdown snapshots for feedback. They incorporate feedback and re-share.
2. **JSON round-trip.** Export a map as JSON, send it to a teammate, they import and extend it, then export and send back. Merge manually by importing and comparing.
3. **Divide and conquer.** Split a large topic into separate maps. Each team member owns their section. Combine results by creating a master map that references each sub-map.
4. **Use descriptions for context.** Write thorough node descriptions so that anyone opening the map later understands the reasoning behind each node, not just its title.
