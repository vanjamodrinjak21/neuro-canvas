# API Overview

The NeuroCanvas API is a RESTful JSON API built on Nuxt 3 server routes. All endpoints live under `/api` and follow consistent conventions for authentication, error handling, and response structure.

## Base URL

| Environment | Base URL |
|---|---|
| Production | `https://neuro-canvas.com/api` |
| Local development | `http://localhost:3000/api` |
| Self-hosted | `https://your-domain.com/api` |

All requests and responses use `Content-Type: application/json` unless noted otherwise (streaming endpoints use `text/event-stream`).

## Authentication

Most endpoints require an authenticated session. NeuroCanvas uses [NextAuth.js](https://next-auth.js.org/) (via `@sidebase/nuxt-auth`) with a JWT session strategy.

### Obtaining a session token

After signing in through one of the available providers (Google, GitHub, Email, or Credentials), the server sets an HTTP-only cookie:

- **Production:** `__Secure-next-auth.session-token`
- **Development:** `next-auth.session-token`

Browser requests automatically include this cookie. For programmatic access, include the cookie header in your request:

```bash
curl -X GET https://neuro-canvas.com/api/user/profile \
  -H "Cookie: __Secure-next-auth.session-token=YOUR_SESSION_TOKEN"
```

### Session validation

Every protected endpoint calls `getServerSession(event)` to validate the session. If the session is missing or expired, the server returns a `401 Unauthorized` response.

Sessions expire after **4 hours** by default and are managed via JWT tokens with a `lastActivity` timestamp for refresh tracking.

## Rate Limiting

API rate limits are enforced using a Redis-backed sliding window counter (INCR + EXPIRE pattern).

| Scope | Limit | Window |
|---|---|---|
| Sync operations | 120 requests | 60 seconds |
| AI endpoints | Per-provider limits apply | Varies |

When a rate limit is exceeded, the server returns `429 Too Many Requests`. The `X-RateLimit-Remaining` header is included on sync responses to indicate remaining quota.

```json
{
  "statusCode": 429,
  "statusMessage": "Rate limit exceeded. Try again later."
}
```

Rate limiting fails open: if Redis is unavailable, requests are allowed through to prevent outages from cascading.

## Response Format

### Successful responses

NeuroCanvas API responses are flat JSON objects. The shape depends on the endpoint, but common patterns include:

**Single resource:**
```json
{
  "id": "clx1abc2d0001",
  "email": "user@example.com",
  "name": "Jane Doe",
  "createdAt": "2025-06-15T10:30:00.000Z"
}
```

**Collection with metadata:**
```json
{
  "maps": [
    { "id": "map_1", "title": "Project Roadmap", "syncVersion": 3 }
  ],
  "deleted": ["map_old_1"]
}
```

**Operation result:**
```json
{
  "success": true,
  "stored": 12
}
```

### Paginated responses

Sync pull endpoints support time-based pagination using the `since` query parameter (ISO 8601 timestamp). The response includes all records updated after the specified time:

```
GET /api/sync/pull?since=2025-06-01T00:00:00.000Z
```

### Error responses

All errors follow the Nuxt/H3 error format with an HTTP status code and a human-readable message:

```json
{
  "statusCode": 400,
  "statusMessage": "Missing required fields: mapId, data, title, checksum"
}
```

AI proxy endpoints return errors with a `message` field instead:

```json
{
  "statusCode": 500,
  "message": "AI request failed"
}
```

## HTTP Status Codes

| Code | Meaning | When used |
|---|---|---|
| `200` | OK | Successful GET, PATCH, POST, or DELETE |
| `201` | Created | Not currently used (successful creation returns 200) |
| `400` | Bad Request | Missing or invalid parameters, validation failure |
| `401` | Unauthorized | Missing or expired session |
| `403` | Forbidden | Valid session but insufficient permissions |
| `404` | Not Found | Resource does not exist |
| `409` | Conflict | Sync version conflict (server has a newer version) or duplicate resource |
| `423` | Locked | Map is locked by another sync operation |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Unexpected server failure |

## CSRF Protection

NextAuth.js includes built-in CSRF protection via a `csrfToken` cookie. For state-mutating requests initiated outside of the standard NextAuth.js flow, the CSRF token is validated automatically through the cookie mechanism.

Cookie names follow the environment convention:

- **Production:** `__Secure-next-auth.csrf-token` (Secure, HttpOnly, SameSite=Lax)
- **Development:** `next-auth.csrf-token`

## Health Check

A public health check endpoint is available without authentication:

```
GET /api/health
```

```json
{
  "status": "ok",
  "timestamp": "2025-06-15T10:30:00.000Z",
  "version": "1.0.0"
}
```

## Caching

NeuroCanvas uses Redis for server-side caching with the following TTL policies:

| Cache target | TTL |
|---|---|
| Map metadata | 1 hour |
| Map data | 1 hour |
| Semantic embeddings | 7 days |
| AI response cache | 1 hour (configurable) |
| AI user memory | 30 days |
| Sync state | 24 hours |

Cache is automatically invalidated on write operations (push, update, delete).
