# Desktop (Tauri)

::callout{type="warning"}
The desktop app is currently in **beta**. Some features may be unstable or incomplete. Data loss is unlikely but not impossible — always keep cloud sync enabled as a backup. Report issues on GitHub.
::

NeuroCanvas Desktop is a native application built with [Tauri 2](https://v2.tauri.app/) and a Rust backend. It provides native performance, local ML inference, and full offline support.

## Why Tauri

Tauri bundles a system webview instead of shipping an entire browser engine (as Electron does). This results in:

- **Smaller binary size** — the NeuroCanvas installer is typically 15-25 MB depending on platform, compared to 150+ MB for Electron-based apps
- **Lower memory usage** — uses the OS-native webview (WebKit on macOS, WebView2 on Windows, WebKitGTK on Linux)
- **Native Rust backend** — CPU-intensive operations (ML inference, file I/O) run in native Rust, not JavaScript

## Installation

### macOS

1. Download the `.dmg` file for your architecture from the [releases page](https://github.com/neurocanvas/neurocanvas/releases).
2. Open the `.dmg` and drag **NeuroCanvas** to your **Applications** folder.
3. On first launch, macOS may block the app. Go to **System Settings > Privacy & Security** and click **Open Anyway**.

Build targets:
- `aarch64-apple-darwin` — Apple Silicon (M1/M2/M3/M4)
- `x86_64-apple-darwin` — Intel
- `universal-apple-darwin` — Universal binary (both architectures)

### Windows

1. Download the `.msi` or `.exe` installer from the releases page.
2. Run the installer and follow the prompts.
3. Windows SmartScreen may show a warning for unsigned builds. Click **More info > Run anyway**.

Windows requires **WebView2** runtime (included in Windows 11, auto-installed on Windows 10).

### Linux

**AppImage (recommended):**
1. Download the `.AppImage` file from the releases page.
2. Make it executable: `chmod +x NeuroCanvas_*.AppImage`
3. Run it: `./NeuroCanvas_*.AppImage`

**Debian/Ubuntu (.deb):**
```bash
sudo dpkg -i neurocanvas_*.deb
sudo apt-get install -f  # Install any missing dependencies
```

Linux requires WebKitGTK 4.1+. On Ubuntu 22.04+:
```bash
sudo apt install libwebkit2gtk-4.1-0
```

## Local ML Inference

The desktop app includes a native ONNX Runtime integration for running ML models locally. This is the primary advantage over the web platform.

### How It Works

The Rust backend loads the `all-MiniLM-L6-v2` sentence transformer model via the [ort](https://github.com/pykeio/ort) crate. Inference runs natively on the CPU (or Apple Neural Engine / CoreML on macOS) without going through JavaScript.

Available ML commands exposed to the frontend:

| Command | Description |
|---------|-------------|
| `ml_init` | Load the ONNX model (quantized or full precision) |
| `ml_embed` | Generate a single text embedding (384 dimensions) |
| `ml_embed_batch` | Batch embed multiple texts with progress events |
| `ml_compute_similarities` | Compute pairwise cosine similarities above a threshold |
| `ml_download_full_model` | Download the full-precision model from HuggingFace |
| `ml_is_full_model_available` | Check if the full model has been downloaded |
| `ml_delete_full_model` | Delete the downloaded full-precision model |

### Model Variants

- **Quantized** (default) — bundled with the app (~30 MB). Faster inference, slightly lower accuracy. Loads on startup.
- **Full precision** — downloaded on demand (~90 MB). Higher accuracy, slower inference. Stored in the app data directory.

On Apple Silicon Macs, the ONNX runtime uses **CoreML** acceleration for faster inference.

### Performance Comparison

| Operation | Web (WASM) | Desktop (Rust/ONNX) |
|-----------|-----------|---------------------|
| Single embedding | ~50ms | ~8ms |
| Batch (100 nodes) | ~3s | ~400ms |
| Similarity (100 nodes) | ~200ms | ~30ms |

*Benchmarks on Apple M2, 16GB RAM. Actual performance varies by hardware.*

## Offline Mode

The desktop app works fully offline for local maps:

- Maps are stored locally in IndexedDB within the webview
- ML inference runs entirely on-device — no API calls needed
- Semantic field visualization, similarity computation, and AI suggestions all work offline
- Authentication is bypassed in desktop mode (the app checks for `window.__TAURI__`)

When internet connectivity is available, maps sync to the server automatically. Offline changes queue and merge on reconnect.

::callout{type="info"}
AI features that require external LLM APIs (map generation from descriptions, advanced insight analysis) still require an internet connection and configured API keys.
::

## Desktop-Specific Features

### Native File Dialogs

The desktop app uses native OS file dialogs for:
- Exporting maps as JSON, PNG, or SVG
- Importing map data from files
- Selecting custom model files

These are powered by the `tauri-plugin-dialog` and `tauri-plugin-fs` plugins.

### System Integration

- **Window management** — resizable window (minimum 800x600), remembers size and position
- **Native menus** — standard OS menu bar on macOS
- **Deep links** — open `neurocanvas://` URLs directly in the app
- **Auto-updater** — checks for updates on launch (when online)

### Hardware Detection

The app detects hardware capabilities on startup to optimize ML performance:

- Apple Silicon detection (for CoreML acceleration)
- Discrete GPU detection (AMD/NVIDIA on macOS)
- Results determine which model variant to recommend

## Building from Source

Prerequisites:
- Node.js 20+
- Rust 1.77+
- Platform-specific dependencies (see [Tauri prerequisites](https://v2.tauri.app/start/prerequisites/))

```bash
# Clone the repository
git clone https://github.com/neurocanvas/neurocanvas.git
cd neurocanvas

# Install dependencies
npm install

# Run in development mode
npm run tauri:dev

# Build for production
npm run tauri:build

# Platform-specific builds
npm run tauri:build:mac-arm       # Apple Silicon
npm run tauri:build:mac-x86       # Intel Mac
npm run tauri:build:mac-universal # Universal binary
npm run tauri:build:win-x86       # Windows x64
npm run tauri:build:win-arm       # Windows ARM
```

The built application is output to `src-tauri/target/release/bundle/`.
