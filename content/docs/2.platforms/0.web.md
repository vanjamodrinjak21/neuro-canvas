# Web Platform

The web app at [neuro-canvas.com](https://neuro-canvas.com) is the primary and most stable way to use NeuroCanvas. It runs entirely in the browser with no installation required.

## Supported Browsers

NeuroCanvas supports the **latest two major versions** of the following browsers:

| Browser | Minimum Version | Status |
|---------|----------------|--------|
| Google Chrome | 120+ | Fully supported |
| Mozilla Firefox | 121+ | Fully supported |
| Safari (macOS) | 17+ | Fully supported |
| Microsoft Edge | 120+ | Fully supported |
| Safari (iOS) | 17+ | Supported (touch optimized) |
| Chrome (Android) | 120+ | Supported (touch optimized) |

NeuroCanvas requires a browser with support for ES2022, WebWorkers, IndexedDB, and the Canvas API. Internet Explorer is not supported.

## Progressive Web App

NeuroCanvas can be installed as a PWA on supported platforms. When installed, it behaves like a native app with its own window, icon, and task bar entry.

To install the PWA:

1. Navigate to [neuro-canvas.com](https://neuro-canvas.com) in Chrome or Edge.
2. Click the install icon in the address bar (or use the browser menu).
3. Confirm the installation prompt.

On iOS Safari, use **Share > Add to Home Screen**.

PWA benefits:
- Launches in its own window without browser chrome
- Offline access to previously loaded maps via IndexedDB (Dexie)
- Background sync when connectivity resumes
- Automatic updates when a new version is deployed

## AI Processing on Web

The web platform runs AI features (semantic embeddings, similarity computation) inside a **Web Worker** using `@huggingface/transformers`. This keeps the main thread responsive while the model runs inference in the background.

The embedding model (`all-MiniLM-L6-v2`) is downloaded and cached in the browser on first use. Subsequent loads are instant.

::callout{type="info"}
The initial model download is approximately 23 MB. After caching, the model loads from IndexedDB in under 2 seconds on most hardware.
::

## Performance Tips

### Hardware Acceleration

- **Enable GPU acceleration** in your browser settings. Chrome: `chrome://flags/#enable-gpu-rasterization`. This significantly improves canvas rendering for large maps.
- NeuroCanvas uses D3.js for force-directed layouts and the HTML Canvas API for rendering. Both benefit from hardware-accelerated compositing.

### Large Maps (100+ nodes)

- **Disable the semantic field layer** when you do not need it. The field visualization computes pairwise similarities across all nodes, which scales quadratically.
- Use **zoom to fit** (keyboard shortcut `F`) to recenter the viewport instead of manually panning across large maps.
- Close the **AI Suggestions Panel** when not actively using it to reduce background computation.
- Consider splitting very large maps (500+ nodes) into multiple linked maps for better performance.

### Memory Management

- Each open browser tab with a map consumes memory for the D3 force simulation, node data, and cached embeddings. Close unused tabs.
- If the browser tab becomes sluggish, refresh the page. Map data is auto-saved to IndexedDB and will restore instantly.

## Mobile Web

NeuroCanvas is fully responsive and optimized for mobile browsers with viewports of **390px and wider**.

Mobile web features:
- **Bottom tab bar** navigation (Home, Search, Maps, Settings) replaces the desktop sidebar on screens 768px and narrower
- **Touch-optimized canvas** with pinch-to-zoom and two-finger pan
- **Floating toolbars** at the top and bottom of the canvas editor
- **Bottom sheet** panels for node properties and AI suggestions
- **44px minimum touch targets** on all interactive elements

::callout{type="warning"}
Mobile web provides the same core functionality as desktop web, but some interactions (multi-select drag, right-click context menus) are adapted for touch input. For the best mobile experience with native gestures, see the [Mobile (Capacitor)](/docs/platforms/mobile-capacitor) platform.
::

## Data Storage

On the web platform, map data is stored in two locations:

1. **Server (PostgreSQL)** — canonical source for authenticated users. Maps are synced automatically.
2. **Browser (IndexedDB via Dexie)** — local cache for offline access and instant loading.

Auto-save triggers every few seconds after changes. If you lose connectivity, changes queue locally and sync when the connection resumes.

## Keyboard Shortcuts

The web platform supports the full set of keyboard shortcuts:

| Shortcut | Action |
|----------|--------|
| `Ctrl/Cmd + N` | Create new node |
| `Ctrl/Cmd + S` | Force save |
| `Ctrl/Cmd + Z` | Undo |
| `Ctrl/Cmd + Shift + Z` | Redo |
| `Delete / Backspace` | Delete selected |
| `F` | Zoom to fit |
| `Ctrl/Cmd + +/-` | Zoom in/out |
| `Space + Drag` | Pan canvas |
