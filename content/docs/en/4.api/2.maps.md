# Maps API

Maps are the core data structure in NeuroCanvas. Each map contains a collection of nodes, edges, camera state, and settings stored as a JSON document. The Maps API handles creation, retrieval, updates, and deletion through a sync-based protocol backed by PostgreSQL and Redis.

## Map Object Schema

Maps are stored in PostgreSQL with the following structure:

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Client-generated CUID, set on creation |
| `userId` | `string` | Owner's user ID |
| `title` | `string` | Map display name |
| `data` | `object` | JSON blob containing nodes, edges, camera, settings, and rootNodeId |
| `checksum` | `string` | SHA-256 hash of the data for integrity verification |
| `preview` | `string?` | Base64-encoded thumbnail image |
| `syncVersion` | `integer` | Monotonically increasing version number for optimistic concurrency |
| `byteSize` | `integer` | Size of the data payload in bytes |
| `tags` | `string[]` | User-defined tags for organization |
| `deletedAt` | `datetime?` | Soft-delete timestamp (null if active) |
| `createdAt` | `datetime` | Creation timestamp |
| `updatedAt` | `datetime` | Last modification timestamp |

### The `data` JSON Structure

The `data` field is a JSON object with the following shape:

```json
{
  "nodes": { ... },
  "edges": { ... },
  "camera": { "x": 0, "y": 0, "zoom": 1 },
  "settings": {
    "gridEnabled": false,
    "gridSize": 20,
    "snapToGrid": false,
    "backgroundColor": "#0A0A0C",
    "defaultNodeStyle": {},
    "defaultEdgeStyle": {}
  },
  "rootNodeId": "node_abc123"
}
```

## Node Object Schema

Nodes represent individual concepts on the canvas.

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique node identifier |
| `type` | `string` | One of: `text`, `image`, `icon`, `note` |
| `position` | `object` | `{ x: number, y: number }` — canvas coordinates |
| `size` | `object` | `{ width: number, height: number }` — pixel dimensions |
| `content` | `string` | Display text or content |
| `style` | `object` | Visual styling (see below) |
| `parentId` | `string?` | Parent node ID for hierarchical relationships |
| `collapsed` | `boolean?` | Whether child nodes are hidden |
| `locked` | `boolean?` | Whether the node is locked from editing |
| `isRoot` | `boolean?` | Marks the main topic node |
| `metadata` | `object?` | Extended metadata including AI descriptions and categories |
| `semantic` | `object?` | Embedding data populated by the AI system |
| `createdAt` | `number` | Unix timestamp (milliseconds) |
| `updatedAt` | `number` | Unix timestamp (milliseconds) |

### Node Style Object

## Edge Object Schema

Edges represent connections between nodes.

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique edge identifier |
| `sourceId` | `string` | ID of the source node |
| `targetId` | `string` | ID of the target node |
| `style` | `object` | Visual styling (see below) |
| `label` | `string?` | Text label on the edge |
| `controlPoints` | `array?` | Bezier control points `[{ x, y }]` |
| `sourceAnchor` | `string?` | Anchor position: `top`, `right`, `bottom`, `left` |
| `targetAnchor` | `string?` | Anchor position: `top`, `right`, `bottom`, `left` |
| `ai` | `object?` | AI metadata (relationship type, confidence, reasoning) |
| `createdAt` | `number` | Unix timestamp (milliseconds) |
| `updatedAt` | `number` | Unix timestamp (milliseconds) |

### Edge Style Object

Allowed `type` values: `bezier`, `straight`, `orthogonal`.

## Sync Endpoints

Maps use a sync-based protocol rather than traditional REST CRUD. The client generates map IDs locally and pushes/pulls data to stay in sync with the server.

### Pull Maps

Retrieve the user's maps from the server.

```
GET /api/sync/pull
```

**Query parameters:**

| Parameter | Type | Description |
|---|---|---|
| `since` | `string` | ISO 8601 timestamp — only return maps updated after this time |
| `mapId` | `string` | Fetch a single map by ID |

**Example: Pull all maps**
```bash
curl -X GET https://neuro-canvas.com/api/sync/pull \
  -H "Cookie: __Secure-next-auth.session-token=..."
```

**Response:**
```json
{
  "maps": [
    {
      "id": "clx1map001",
      "title": "Project Roadmap",
      "data": { "nodes": {}, "edges": {}, "camera": { "x": 0, "y": 0, "zoom": 1 } },
      "syncVersion": 5,
      "checksum": "a1b2c3d4e5f6...",
      "tags": ["work", "planning"],
      "preview": "data:image/png;base64,...",
      "byteSize": 4096,
      "createdAt": "2026-01-01T10:00:00.000Z",
      "updatedAt": "2026-01-15T10:30:00.000Z"
    }
  ],
  "deleted": ["clx1mapOld"]
}
```

**Example: Pull changes since last sync**
```bash
curl -X GET "https://neuro-canvas.com/api/sync/pull?since=2026-01-10T00:00:00.000Z" \
  -H "Cookie: __Secure-next-auth.session-token=..."
```

**Example: Pull a single map**
```bash
curl -X GET "https://neuro-canvas.com/api/sync/pull?mapId=clx1map001" \
  -H "Cookie: __Secure-next-auth.session-token=..."
```

The `deleted` array contains IDs of maps that were soft-deleted since the `since` timestamp, allowing the client to remove them locally.

### Push Map

Create or update a map on the server. Uses upsert semantics — if the map ID does not exist, it is created; otherwise it is updated.

```
POST /api/sync/push
Content-Type: application/json

{
  "mapId": "clx1map001",
  "title": "Project Roadmap",
  "data": {
    "nodes": {},
    "edges": {},
    "camera": { "x": 0, "y": 0, "zoom": 1 },
    "settings": {},
    "rootNodeId": "node_root"
  },
  "syncVersion": 5,
  "checksum": "a1b2c3d4e5f6...",
  "deviceId": "device_abc",
  "preview": "data:image/png;base64,...",
  "tags": ["work", "planning"]
}
```

**Required fields:** `mapId`, `title`, `data`, `checksum`

**Response (success):**
```json
{
  "ok": true,
  "syncVersion": 6
}
```

**Response (no changes):**
```json
{
  "ok": true,
  "syncVersion": 5,
  "noChange": true
}
```

**Errors:**

| Status | Condition |
|---|---|
| `400` | Missing required fields |
| `409` | Version conflict — server has a newer version |
| `423` | Map is locked by another sync operation |
| `429` | Rate limit exceeded |

### Delete Map (Soft Delete)

Maps are soft-deleted by sending a push with `action: "delete"`. This sets the `deletedAt` timestamp and publishes the deletion to other connected devices.

```
POST /api/sync/push
Content-Type: application/json

{
  "mapId": "clx1map001",
  "title": "",
  "data": {},
  "checksum": "",
  "action": "delete",
  "deviceId": "device_abc"
}
```

**Response:**
```json
{
  "ok": true
}
```

## Sync Protocol Details

### Optimistic Concurrency

Each map has a `syncVersion` that increments with every push. The server compares the client's `syncVersion` against the current database version:

- If the client version is **less than** the server version, a `409 Conflict` is returned
- If the checksums match, the push is skipped and `noChange: true` is returned
- Otherwise, the version is incremented and the data is saved

### Distributed Locking

Before writing, the server acquires a short-lived distributed lock on the map ID. If the lock is held by another operation, a `423 Locked` response is returned.

### Version History

The server retains recent versions of each map. Older versions are pruned automatically. Each version records the data snapshot, checksum, device ID, and timestamp.

### Real-Time Notifications

After a successful push, connected clients receive live updates via the SSE stream endpoint when maps are updated or deleted from other devices.

### Sync Stream

```
GET /api/sync/stream
```

Server-Sent Events endpoint for real-time sync notifications. Clients receive events when maps are updated or deleted from other devices.

### Sync Status

```
GET /api/sync/status
```

Returns the user's current sync state including last sync time, device ID, and map count.

## Map Sharing

### Create or Get Share Link

Generate a shareable token for a map, or retrieve an existing one.

```
POST /api/maps/:id/share
```

**Response:**
```json
{
  "token": "abc123def456",
  "url": "https://neuro-canvas.com/share/abc123def456",
  "expiresAt": null
}
```

### View Shared Map (Public)

Retrieve map data via a share token. No authentication required.

```
GET /api/share/:token
```

**Response:** The full map data (nodes, edges, camera, settings) in read-only format. Returns `404` if the token is invalid or expired.

### Map Preview (OG Image)

Generate or retrieve a PNG preview of a map, used for OpenGraph meta images.

```
GET /api/maps/:id/render
```

**Response:** A PNG image of the map canvas.

## Caching Strategy

Map data is cached in Redis for fast retrieval:

- **Map metadata** (`map:meta:{mapId}`): `syncVersion` and `checksum`, 1-hour TTL
- **Map data** (`map:data:{mapId}`): Full data payload with title and tags, 1-hour TTL

The pull endpoint checks Redis first and falls back to PostgreSQL. After a database read, the result is written back to Redis to warm the cache.

On push, both the metadata and data caches are immediately updated to ensure subsequent reads are fast and consistent.
