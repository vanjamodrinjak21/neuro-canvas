# Mobile (Capacitor)

::callout{type="warning"}
The mobile app is currently in **beta**. Touch interactions and UI are functional but still being refined. Some AI features are limited compared to the web and desktop platforms. Report issues on GitHub.
::

NeuroCanvas Mobile is built with [Capacitor 6](https://capacitorjs.com/) to deliver a native iOS and Android experience from the same Vue.js codebase that powers the web app.

## Platform Support

| Platform | Minimum Version | Status |
|----------|----------------|--------|
| iOS | 16.0+ | Beta |
| Android | API 26 (Android 8.0+) | Beta |

## Architecture

Capacitor wraps the NeuroCanvas Nuxt 3 SPA inside a native WebView, with bridge access to native device APIs. The app is compiled as a static SPA (`nuxt generate`) and bundled into the native project.

The native shell provides:
- **Native splash screen** — dark branded screen (#032221) with a 2-second display
- **Status bar integration** — dark style matching the app theme
- **Keyboard handling** — automatic body resize when the soft keyboard appears
- **Safe area insets** — automatic content inset handling on iOS (notch, Dynamic Island)

## Touch-Optimized Canvas

The canvas editor is redesigned for touch input:

### Gestures

| Gesture | Action |
|---------|--------|
| Single tap | Select node |
| Double tap | Edit node label |
| Long press | Open context menu |
| Single finger drag | Move selected node |
| Two-finger pinch | Zoom in/out |
| Two-finger pan | Pan the canvas |
| Two-finger tap | Undo last action |

### Touch Targets

All interactive elements maintain a minimum touch target of **44x44 points** (Apple HIG) / **48x48 dp** (Material Design), including toolbar buttons, node handles, and menu items.

## Mobile-Specific UI

The mobile interface adapts significantly from the desktop layout:

### Bottom Tab Bar

On screens 768px and narrower, the desktop sidebar is replaced by a bottom tab bar with four tabs:

- **Home** — dashboard with recent maps and quick actions
- **Search** — search across all maps and nodes
- **Maps** — list view of all maps (single column, not grid)
- **Settings** — app preferences and account management

### Floating Toolbars

In the canvas editor:
- **Top toolbar** — map title, undo/redo, share button
- **Bottom toolbar** — node creation, connection mode, AI tools
- Both toolbars float above the canvas and collapse on scroll to maximize canvas space

### Bottom Sheets

Panels that appear as sidebars on desktop (node properties, AI suggestions) are presented as **bottom sheets** on mobile:
- Swipe up to expand to full height
- Swipe down to collapse or dismiss
- Backdrop tap to close

### List View for Maps

The "All Maps" page uses a single-column list layout instead of the grid layout used on desktop. Each list item shows the map title, last edited date, node count, and a thumbnail preview.

## Native Device Features

Capacitor provides access to native APIs through plugins:

| Feature | Plugin | Description |
|---------|--------|-------------|
| File system | `@capacitor/filesystem` | Save/load map exports to device storage |
| Share | `@capacitor/share` | Native share sheet for exporting maps |
| Haptics | `@capacitor/haptics` | Tactile feedback on node creation, deletion, and connections |
| Dialog | `@capacitor/dialog` | Native alert and confirm dialogs |

### Haptic Feedback

The app uses haptic feedback for:
- **Light** — node selection, toolbar button taps
- **Medium** — node creation, edge connection completed
- **Heavy** — node deletion, destructive actions

## AI Features on Mobile

### What Works

- Semantic field visualization (computed in web worker)
- Node similarity detection
- AI-generated map suggestions
- Insight panel analysis

### Known Limitations

- **Embedding computation is slower** — mobile devices have less CPU headroom than desktops. Batch embedding of large maps (50+ nodes) may take noticeably longer.
- **No local ONNX inference** — unlike the Tauri desktop app, the mobile app runs embeddings in a JavaScript Web Worker using `@huggingface/transformers` (WASM). There is no native Rust ML backend on mobile.
- **Memory constraints** — on devices with 4GB RAM or less, very large maps (200+ nodes) with the semantic field layer enabled may cause the WebView to reload. Disable the semantic field layer for better stability on low-memory devices.
- **Background processing** — iOS aggressively suspends background WebViews. Long-running AI operations may be interrupted if the app is backgrounded.

### Development Builds

To build and run the mobile app locally:

**Prerequisites:**
- Node.js 20+
- For iOS: macOS with Xcode 15+, CocoaPods
- For Android: Android Studio with SDK 26+

```bash
# Install dependencies
npm install

# Generate the static SPA
npm run generate

# Sync native projects
npm run cap:sync

# Run on iOS simulator or device
npm run cap:ios

# Run on Android emulator or device
npm run cap:android
```

### iOS Development

```bash
# Open the iOS project in Xcode
npx cap open ios
```

The iOS project uses the scheme name `NeuroCanvas` with automatic content inset handling.

### Android Development

```bash
# Open the Android project in Android Studio
npx cap open android
```

The Android project targets API 26+ with mixed content disabled for security and web contents debugging enabled for development builds.

## Configuration

The Capacitor configuration (`capacitor.config.ts`) defines platform-specific settings:

- **App ID:** `com.neurocanvas.app`
- **Web directory:** `.output/public` (Nuxt static output)
- **iOS background color:** `#032221`
- **Android background color:** `#032221`
- **Splash screen:** 2-second branded splash with auto-hide

For development with hot reload, uncomment the `server.url` option in `capacitor.config.ts` and point it to your local dev server:

```typescript
server: {
  url: 'http://YOUR_LOCAL_IP:3000',
  cleartext: true
}
```

::callout{type="info"}
Use your machine's local network IP (not `localhost`) so the mobile device or emulator can reach the dev server over the network.
::
