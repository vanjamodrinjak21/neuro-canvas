# Autentifikacija

NeuroCanvas koristi [NextAuth.js](https://next-auth.js.org/) (integriran putem `@sidebase/nuxt-auth`) za autentifikaciju. Sustav podržava više pružatelja i koristi sesije temeljene na JWT-u pohranjene u PostgreSQL bazi podataka putem Prisma adaptera.

## Pregled tijeka autentifikacije

```
User clicks "Sign In"
    |
    v
Provider selection (Google, GitHub, Email, or Credentials)
    |
    v
OAuth redirect / Credentials validation / Magic link
    |
    v
NextAuth callback -> JWT created -> Session cookie set
    |
    v
User redirected to /dashboard (new users) or previous page
```

### Strategija sesija

NeuroCanvas koristi sesije temeljene na JWT-u pohranjene u sigurnim HTTP-only kolačićima. Sesije automatski istječu i obnavljaju se za vrijeme aktivnog korištenja.

### Adapter baze podataka

Adapter baze podataka trajno pohranjuje korisnike, račune i tokene za provjeru. Ovo omogućuje:

- Povezivanje više OAuth pružatelja s jednim korisničkim računom
- Sigurnu pohranu lozinki
- Upravljanje tokenima za provjeru e-pošte

## Dostupni pružatelji

### Google OAuth

Prijava s Googleom koristeći OAuth 2.0 autorizacijski kod tijek.

- Zahtijeva varijable okruženja `GOOGLE_CLIENT_ID` i `GOOGLE_CLIENT_SECRET`
- Zahtijeva izvanmrežni pristup s `access_type: "offline"` za tokene osvježavanja
- Omogućeno je povezivanje računa e-poštom (`allowDangerousEmailAccountLinking: true`)

### GitHub OAuth

Prijava s GitHubom koristeći OAuth.

- Zahtijeva varijable okruženja `GITHUB_CLIENT_ID` i `GITHUB_CLIENT_SECRET`
- Zahtijeva opsege `read:user` i `user:email`
- Automatski dohvaća primarnu e-poštu iz GitHub emails API-ja ako nije uključena u profil
- Omogućeno je povezivanje računa e-poštom

### E-pošta (čarobna veza)

Prijava bez lozinke putem e-pošte koristeći [Resend](https://resend.com/) za isporuku transakcijske e-pošte.

- Zahtijeva `RESEND_API_KEY` i konfiguraciju pošiljatelja e-pošte
- Šalje brendiranu HTML e-poštu s vezom za prijavu jednim klikom
- Veze istječu nakon 24 sata
- Dostupno samo kada je Resend konfiguriran

### Vjerodajnice (e-pošta + lozinka)

Tradicionalna autentifikacija e-poštom i lozinkom.

- Uvijek dostupna bez obzira na konfiguraciju okruženja
- Lozinke su sigurno hashirane prije pohrane
- Lozinke moraju zadovoljiti minimalne zahtjeve snage (duljina, raznolikost znakova)
- Lozinke se provjeravaju u poznatim bazama podataka probijenih podataka pri registraciji i promjeni lozinke
- Ako je dvofaktorska autentifikacija omogućena, pri prijavi je potreban valjani TOTP kod

## API krajnje točke

### Dobivanje trenutne sesije

Dohvatite sesiju trenutno autentificiranog korisnika.

```
GET /api/auth/session
```

**Odgovor (autentificiran):**
```json
{
  "user": {
    "id": "clx1abc2d0001",
    "email": "user@example.com",
    "name": "Jane Doe",
    "image": "https://lh3.googleusercontent.com/..."
  },
  "expires": "2026-01-15T14:30:00.000Z"
}
```

**Odgovor (nije autentificiran):**
```json
{}
```

### Popis dostupnih pružatelja

Vraća pružatelje autentifikacije konfigurirane na poslužitelju.

```
GET /api/auth/providers
```

**Odgovor:**
```json
{
  "google": {
    "id": "google",
    "name": "Google",
    "type": "oauth",
    "signinUrl": "/api/auth/signin/google",
    "callbackUrl": "/api/auth/callback/google"
  },
  "github": {
    "id": "github",
    "name": "GitHub",
    "type": "oauth",
    "signinUrl": "/api/auth/signin/github",
    "callbackUrl": "/api/auth/callback/github"
  },
  "credentials": {
    "id": "credentials",
    "name": "Credentials",
    "type": "credentials",
    "signinUrl": "/api/auth/signin/credentials",
    "callbackUrl": "/api/auth/callback/credentials"
  }
}
```

Odgovor uključuje samo pružatelje čije su varijable okruženja konfigurirane. Na primjer, Google se neće pojaviti ako `GOOGLE_CLIENT_ID` nedostaje.

### Prijava

Pokrenite tijek prijave. Metoda ovisi o vrsti pružatelja.

#### OAuth pružatelji (Google, GitHub)

```
POST /api/auth/signin/:provider
```

Preusmjerava korisnika na stranicu za autorizaciju OAuth pružatelja. Nakon pristanka, korisnik se preusmjerava natrag na `/api/auth/callback/:provider`, što dovršava JWT razmjenu i postavlja kolačić sesije.

#### Pružatelj vjerodajnica

```
POST /api/auth/signin/credentials
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "SecureP4ss"
}
```

**Uspjeh:** Postavlja kolačić sesije i preusmjerava na `/dashboard`.

**Neuspjeh (401):**
```json
{
  "error": "CredentialsSignin",
  "url": "/auth/error?error=CredentialsSignin"
}
```

### Registracija (Vjerodajnice)

Stvorite novi račun s e-poštom i lozinkom. Ovo je prilagođena krajnja točka izvan NextAuth.js tijeka.

```
POST /api/auth/register
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "SecureP4ss",
  "name": "Jane Doe"
}
```

**Uspjeh (200):**
```json
{
  "success": true,
  "user": {
    "id": "clx1abc2d0001",
    "email": "user@example.com",
    "name": "Jane Doe",
    "createdAt": "2026-01-15T10:30:00.000Z"
  }
}
```

**Greške:**

| Status | Uvjet |
|---|---|
| `400` | Nedostaje e-pošta ili lozinka |
| `400` | Nevaljani format e-pošte |
| `400` | Lozinka ne zadovoljava zahtjeve |
| `400` | Lozinka pronađena u poznatim probijenim podacima |
| `409` | Račun s ovom e-poštom već postoji |
| `429` | Prekoračeno ograničenje brzine |

### Odjava

Prekinite trenutnu sesiju.

```
POST /api/auth/signout
```

Briše kolačić sesije i preusmjerava na `/auth/signin`.

### CSRF token

Dohvatite CSRF token za upotrebu u zahtjevima obrazaca.

```
GET /api/auth/csrf
```

**Odgovor:**
```json
{
  "csrfToken": "abc123..."
}
```

## Upotreba sesija u API zahtjevima

Sve zaštićene serverske rute provjeravaju sesiju koristeći `getServerSession(event)` iz modula `#auth`:

```typescript
import { getServerSession } from '#auth'

export default defineEventHandler(async (event) => {
  const session = await getServerSession(event)

  if (!session?.user?.email) {
    throw createError({
      statusCode: 401,
      statusMessage: 'Unauthorized'
    })
  }

  // Access session.user.id, session.user.email, etc.
})
```

Kolačić sesije automatski se uključuje u zahtjeve preglednika. Za korištenje web aplikacije nije potreban ručni zaglavlje `Authorization`.

## Krajnje točke korisničkog profila

### Dohvati profil

```
GET /api/user/profile
```

Vraća profil autentificiranog korisnika zajedno s njegovim povezanim OAuth pružateljima.

**Odgovor:**
```json
{
  "id": "clx1abc2d0001",
  "email": "user@example.com",
  "name": "Jane Doe",
  "bio": "Researcher and mind mapping enthusiast",
  "image": "https://lh3.googleusercontent.com/...",
  "emailVerified": "2026-01-01T00:00:00.000Z",
  "totpEnabled": false,
  "createdAt": "2026-01-01T00:00:00.000Z",
  "updatedAt": "2026-01-15T10:30:00.000Z",
  "providers": ["google", "github"]
}
```

### Ažuriranje profila

```
PATCH /api/user/profile
Content-Type: application/json
X-Requested-With: XMLHttpRequest

{
  "name": "Jane Smith",
  "bio": "Researcher and mind mapping enthusiast",
  "image": "https://example.com/avatar.jpg"
}
```

Sva polja su neobavezna. Ime je ograničeno na 100 znakova. Bio je ograničen na 280 znakova. Slika mora biti HTTPS URL.

**Odgovor:**
```json
{
  "id": "clx1abc2d0001",
  "email": "user@example.com",
  "name": "Jane Smith",
  "bio": "Researcher and mind mapping enthusiast",
  "image": "https://example.com/avatar.jpg",
  "updatedAt": "2026-01-15T11:00:00.000Z"
}
```

### Promjena lozinke

Promijenite lozinku za račun temeljen na vjerodajnicama. Zahtijeva trenutnu lozinku za provjeru. Nova lozinka se provjerava u HaveIBeenPwned bazi. Promjena lozinke poništava sve postojeće sesije.

```
POST /api/user/change-password
Content-Type: application/json
X-Requested-With: XMLHttpRequest

{
  "currentPassword": "OldP4ssword",
  "newPassword": "NewSecureP4ss"
}
```

**Uspjeh (200):**
```json
{
  "success": true,
  "message": "Password changed successfully"
}
```

**Greške:**

| Status | Uvjet |
|---|---|
| `400` | Nedostaje trenutna ili nova lozinka |
| `400` | Nova lozinka ne zadovoljava zahtjeve |
| `400` | Nova lozinka pronađena u poznatim probijenim podacima |
| `401` | Trenutna lozinka je netočna |
| `403` | Račun ne koristi autentifikaciju lozinkom |
| `429` | Prekoračeno ograničenje brzine |

### Izvoz podataka računa

Preuzmite sve korisničke podatke kao JSON datoteku.

```
GET /api/user/export
```

**Odgovor:** Preuzimanje JSON datoteke koja sadrži podatke profila, sve mape (s čvorovima, vezama i metapodacima), objavljene predloške i metapodatke vjerodajnica (samo nazivi pružatelja — bez šifriranih ključeva).

Ova krajnja točka je ograničena brzinom.

### Dvofaktorska autentifikacija (TOTP)

#### Postavljanje 2FA

Pokrenite postupak postavljanja TOTP-a. Vraća QR kod i tajni ključ za registraciju aplikacije za autentifikaciju.

```
POST /api/user/totp/setup
X-Requested-With: XMLHttpRequest
```

**Odgovor:**
```json
{
  "qrCode": "data:image/png;base64,...",
  "uri": "otpauth://totp/NeuroCanvas:user@example.com?secret=..."
}
```

#### Provjera i omogućivanje 2FA

Provjerite TOTP kod iz aplikacije za autentifikaciju za aktivaciju 2FA. Vraća jednokratne kodove za oporavak.

```
POST /api/user/totp/verify
Content-Type: application/json
X-Requested-With: XMLHttpRequest

{
  "code": "123456"
}
```

**Odgovor:**
```json
{
  "success": true,
  "backupCodes": ["...", "..."]
}
```

::callout{type="warning"}
Pohranite kodove za oporavak na sigurno mjesto. Svaki kod može se koristiti samo jednom. Ako izgubite pristup aplikaciji za autentifikaciju i kodovima za oporavak, bit ćete zaključani iz svog računa.
::

#### Onemogućivanje 2FA

Onemogućite dvofaktorsku autentifikaciju. Zahtijeva ponovnu autentifikaciju lozinkom.

```
POST /api/user/totp/disable
Content-Type: application/json
X-Requested-With: XMLHttpRequest

{
  "password": "YourP4ssword"
}
```

**Odgovor:**
```json
{
  "success": true,
  "message": "Two-factor authentication disabled"
}
```

### Brisanje računa

Trajno brisanje autentificiranog korisničkog računa i svih pridruženih podataka (mape, sesije, OAuth računi). Ova radnja kaskadira kroz sve povezane Prisma modele.

```
DELETE /api/user/account
Content-Type: application/json

{
  "confirmation": "DELETE MY ACCOUNT"
}
```

Točan niz `"DELETE MY ACCOUNT"` mora biti naveden kao potvrda. Bilo koja druga vrijednost vraća grešku `400`.

**Odgovor:**
```json
{
  "success": true,
  "message": "Account deleted successfully"
}
```

## Prilagođene stranice autentifikacije

NeuroCanvas koristi prilagođene stranice autentifikacije umjesto zadanog NextAuth.js korisničkog sučelja:

| Stranica | Putanja |
|---|---|
| Prijava | `/auth/signin` |
| Odjava | `/auth/signin` (preusmjerava ovdje) |
| Greška | `/auth/error` |
| Provjera e-pošte | `/auth/verify` |
| Novi korisnik | `/dashboard` (preusmjerava ovdje) |

## Sigurnosne napomene

- Svi kolačići sesija koriste zastavice `HttpOnly`, `SameSite` i `Secure` u produkciji
- Varijabla okruženja `AUTH_SECRET` je obavezna i mora biti snažan slučajan niz
- OAuth tokeni se pohranjuju šifrirani u bazi podataka
- Lozinke su sigurno hashirane prije pohrane
- Lozinke se provjeravaju u poznatim bazama probijenih podataka pri registraciji i promjeni lozinke
- Promjena lozinke poništava postojeće sesije
- Dvofaktorska autentifikacija koristi standardni TOTP s jednokratnim kodovima za oporavak
- CSRF zaštita se primjenjuje na svim krajnjim točkama koje mijenjaju stanje
- API ključevi u trezoru vjerodajnica su šifrirani na strani poslužitelja
