# API mapa

Mape su temeljna podatkovna struktura u NeuroCanvasu. Svaka mapa sadrži kolekciju čvorova, veza, stanja kamere i postavki pohranjenih kao JSON dokument. API mapa rukuje stvaranjem, dohvaćanjem, ažuriranjem i brisanjem putem protokola temeljenog na sinkronizaciji koji se oslanja na PostgreSQL i Redis.

## Shema objekta mape

Mape su pohranjene u PostgreSQLu sa sljedećom strukturom:

| Polje | Vrsta | Opis |
|---|---|---|
| `id` | `string` | CUID generiran od klijenta, postavlja se pri stvaranju |
| `userId` | `string` | ID korisnika vlasnika |
| `title` | `string` | Prikazno ime mape |
| `data` | `object` | JSON blob koji sadrži čvorove, veze, kameru, postavke i rootNodeId |
| `checksum` | `string` | SHA-256 hash podataka za provjeru integriteta |
| `preview` | `string?` | Minijaturna slika kodirana u Base64 |
| `syncVersion` | `integer` | Monotono rastući broj verzije za optimistično upravljanje istovremenim pristupom |
| `byteSize` | `integer` | Veličina korisnog tereta podataka u bajtovima |
| `tags` | `string[]` | Korisnički definirane oznake za organizaciju |
| `deletedAt` | `datetime?` | Vremenska oznaka mekog brisanja (null ako je aktivna) |
| `createdAt` | `datetime` | Vremenska oznaka stvaranja |
| `updatedAt` | `datetime` | Vremenska oznaka posljednje izmjene |

### JSON struktura polja `data`

Polje `data` je JSON objekt sljedećeg oblika:

```json
{
  "nodes": { ... },
  "edges": { ... },
  "camera": { "x": 0, "y": 0, "zoom": 1 },
  "settings": {
    "gridEnabled": false,
    "gridSize": 20,
    "snapToGrid": false,
    "backgroundColor": "#0A0A0C",
    "defaultNodeStyle": {},
    "defaultEdgeStyle": {}
  },
  "rootNodeId": "node_abc123"
}
```

## Shema objekta čvora

Čvorovi predstavljaju pojedinačne koncepte na platnu.

| Polje | Vrsta | Opis |
|---|---|---|
| `id` | `string` | Jedinstveni identifikator čvora |
| `type` | `string` | Jedno od: `text`, `image`, `icon`, `note` |
| `position` | `object` | `{ x: number, y: number }` — koordinate platna |
| `size` | `object` | `{ width: number, height: number }` — dimenzije u pikselima |
| `content` | `string` | Prikazni tekst ili sadržaj |
| `style` | `object` | Vizualno oblikovanje (vidi dolje) |
| `parentId` | `string?` | ID roditeljskog čvora za hijerarhijske odnose |
| `collapsed` | `boolean?` | Jesu li podređeni čvorovi skriveni |
| `locked` | `boolean?` | Je li čvor zaključan od uređivanja |
| `isRoot` | `boolean?` | Označava čvor glavne teme |
| `metadata` | `object?` | Prošireni metapodaci uključujući AI opise i kategorije |
| `semantic` | `object?` | Podaci ugradnje koje popunjava AI sustav |
| `createdAt` | `number` | Unix vremenska oznaka (milisekunde) |
| `updatedAt` | `number` | Unix vremenska oznaka (milisekunde) |

### Objekt stila čvora

```json
{
  "shape": "rounded",
  "fillColor": "#111113",
  "borderColor": "#27272A",
  "borderWidth": 1.5,
  "textColor": "#FAFAFA",
  "fontSize": 14,
  "fontWeight": 500,
  "shadowEnabled": false,
  "glowEnabled": false
}
```

Dopuštene vrijednosti `shape`: `rounded`, `rectangle`, `circle`, `diamond`.

## Shema objekta veze

Veze predstavljaju uspostavljene veze između čvorova.

| Polje | Vrsta | Opis |
|---|---|---|
| `id` | `string` | Jedinstveni identifikator veze |
| `sourceId` | `string` | ID izvorišnog čvora |
| `targetId` | `string` | ID odredišnog čvora |
| `style` | `object` | Vizualno oblikovanje (vidi dolje) |
| `label` | `string?` | Tekstualna oznaka na vezi |
| `controlPoints` | `array?` | Bezierove kontrolne točke `[{ x, y }]` |
| `sourceAnchor` | `string?` | Pozicija sidra: `top`, `right`, `bottom`, `left` |
| `targetAnchor` | `string?` | Pozicija sidra: `top`, `right`, `bottom`, `left` |
| `ai` | `object?` | AI metapodaci (vrsta odnosa, pouzdanost, obrazloženje) |
| `createdAt` | `number` | Unix vremenska oznaka (milisekunde) |
| `updatedAt` | `number` | Unix vremenska oznaka (milisekunde) |

### Objekt stila veze

```json

  "strokeColor": "#3A3A42",
  "strokeWidth": 2,
  "arrowStart": "none",
  "arrowEnd": "arrow",
```

Dopuštene vrijednosti `type`: `bezier`, `straight`, `orthogonal`.
Dopuštene vrijednosti strelica: `none`, `arrow`, `dot`, `diamond`.

## Krajnje točke sinkronizacije

Mape koriste protokol temeljen na sinkronizaciji umjesto tradicionalnog REST CRUD-a. Klijent lokalno generira ID-ove mapa i gura/povlači podatke kako bi ostao sinkroniziran s poslužiteljem.

### Povlačenje mapa

Dohvatite korisnikove mape s poslužitelja.

```
GET /api/sync/pull
```

**Parametri upita:**

| Parametar | Vrsta | Opis |
|---|---|---|
| `since` | `string` | ISO 8601 vremenska oznaka — vraća samo mape ažurirane nakon ovog vremena |
| `mapId` | `string` | Dohvati jednu mapu prema ID-u |

**Primjer: Povlačenje svih mapa**
```bash
curl -X GET https://neuro-canvas.com/api/sync/pull \
  -H "Cookie: __Secure-next-auth.session-token=..."
```

**Odgovor:**
```json
{
  "maps": [
    {
      "id": "clx1map001",
      "title": "Project Roadmap",
      "data": { "nodes": {}, "edges": {}, "camera": { "x": 0, "y": 0, "zoom": 1 } },
      "syncVersion": 5,
      "checksum": "a1b2c3d4e5f6...",
      "tags": ["work", "planning"],
      "preview": "data:image/png;base64,...",
      "byteSize": 4096,
      "createdAt": "2026-01-01T10:00:00.000Z",
      "updatedAt": "2026-01-15T10:30:00.000Z"
    }
  ],
  "deleted": ["clx1mapOld"]
}
```

**Primjer: Povlačenje promjena od posljednje sinkronizacije**
```bash
curl -X GET "https://neuro-canvas.com/api/sync/pull?since=2026-01-10T00:00:00.000Z" \
  -H "Cookie: __Secure-next-auth.session-token=..."
```

**Primjer: Povlačenje jedne mape**
```bash
curl -X GET "https://neuro-canvas.com/api/sync/pull?mapId=clx1map001" \
  -H "Cookie: __Secure-next-auth.session-token=..."
```

Niz `deleted` sadrži ID-ove mapa koje su meko izbrisane od `since` vremenske oznake, omogućujući klijentu da ih lokalno ukloni.

### Guranje mape

Stvorite ili ažurirajte mapu na poslužitelju. Koristi semantiku upserta — ako ID mape ne postoji, stvara se; inače se ažurira.

```
POST /api/sync/push
Content-Type: application/json

{
  "mapId": "clx1map001",
  "title": "Project Roadmap",
  "data": {
    "nodes": {},
    "edges": {},
    "camera": { "x": 0, "y": 0, "zoom": 1 },
    "settings": {},
    "rootNodeId": "node_root"
  },
  "syncVersion": 5,
  "checksum": "a1b2c3d4e5f6...",
  "deviceId": "device_abc",
  "preview": "data:image/png;base64,...",
  "tags": ["work", "planning"]
}
```

**Obavezna polja:** `mapId`, `title`, `data`, `checksum`

**Odgovor (uspjeh):**
```json
{
  "ok": true,
  "syncVersion": 6
}
```

**Odgovor (nema promjena):**
```json
{
  "ok": true,
  "syncVersion": 5,
  "noChange": true
}
```

**Greške:**

| Status | Uvjet |
|---|---|
| `400` | Nedostaju obavezna polja |
| `409` | Konflikt verzije — poslužitelj ima noviju verziju |
| `423` | Mapa je zaključana drugom operacijom sinkronizacije |
| `429` | Prekoračeno ograničenje brzine |

### Brisanje mape (meko brisanje)

Mape se meko brišu slanjem pusha s `action: "delete"`. Ovo postavlja vremensku oznaku `deletedAt` i objavljuje brisanje na druge povezane uređaje.

```
POST /api/sync/push
Content-Type: application/json

{
  "mapId": "clx1map001",
  "title": "",
  "data": {},
  "checksum": "",
  "action": "delete",
  "deviceId": "device_abc"
}
```

**Odgovor:**
```json
{
  "ok": true
}
```

## Detalji protokola sinkronizacije

### Optimistično upravljanje istovremenim pristupom

Svaka mapa ima `syncVersion` koji se povećava sa svakim pushom. Poslužitelj uspoređuje `syncVersion` klijenta s trenutnom verzijom u bazi podataka:

- Ako je verzija klijenta **manja od** verzije poslužitelja, vraća se `409 Conflict`
- Ako se checksumi podudaraju, push se preskače i vraća `noChange: true`
- Inače se verzija povećava i podaci se pohranjuju

### Distribuirano zaključavanje

Prije pisanja, poslužitelj stječe kratkotrajno distribuirano zaključavanje na ID mape. Ako zaključavanje drži druga operacija, vraća se odgovor `423 Locked`.

### Povijest verzija

Poslužitelj zadržava nedavne verzije svake mape. Starije verzije automatski se čiste. Svaka verzija bilježi snimak podataka, checksum, ID uređaja i vremensku oznaku.

### Obavijesti u stvarnom vremenu

Nakon uspješnog pusha, povezani klijenti primaju ažuriranja uživo putem SSE stream krajnje točke kada se mape ažuriraju ili brišu s drugih uređaja.

### Tok sinkronizacije

```
GET /api/sync/stream
```

Krajnja točka Server-Sent Events za obavijesti o sinkronizaciji u stvarnom vremenu. Klijenti primaju događaje kada se mape ažuriraju ili brišu s drugih uređaja.

### Status sinkronizacije

```
GET /api/sync/status
```

Vraća trenutno stanje sinkronizacije korisnika uključujući vrijeme zadnje sinkronizacije, ID uređaja i broj mapa.

## Dijeljenje mapa

### Stvaranje ili dohvaćanje veze za dijeljenje

Generirajte token za dijeljenje mape ili dohvatite postojeći.

```
POST /api/maps/:id/share
```

**Odgovor:**
```json
{
  "token": "abc123def456",
  "url": "https://neuro-canvas.com/share/abc123def456",
  "expiresAt": null
}
```

### Pregled dijeljene mape (javno)

Dohvatite podatke mape putem tokena za dijeljenje. Autentifikacija nije potrebna.

```
GET /api/share/:token
```

**Odgovor:** Potpuni podaci mape (čvorovi, veze, kamera, postavke) u formatu samo za čitanje. Vraća `404` ako je token nevaljani ili istekao.

### Pregled mape (OG slika)

Generirajte ili dohvatite PNG pregled mape koji se koristi za OpenGraph meta slike.

```
GET /api/maps/:id/render
```

**Odgovor:** PNG slika platna mape.

## Strategija predmemoriranja

Podaci mape predmemorirani su u Redisu za brzo dohvaćanje:

- **Metapodaci mape** (`map:meta:{mapId}`): `syncVersion` i `checksum`, TTL od 1 sata
- **Podaci mape** (`map:data:{mapId}`): Potpuni korisni teret podataka s naslovom i oznakama, TTL od 1 sata

Krajnja točka za povlačenje prvo provjerava Redis i pada natrag na PostgreSQL. Nakon čitanja iz baze podataka, rezultat se zapisuje natrag u Redis radi zagrijavanja predmemorije.

Pri guranju, metapodaci i predmemorija podataka odmah se ažuriraju kako bi se osiguralo da su naknadna čitanja brza i dosljedna.
