# Implementacija s Dockerom

NeuroCanvas se može samostalno postavljati na vlastitoj infrastrukturi pomoću Dockera. Na taj način imate potpunu kontrolu nad svojim podacima, pružateljima autentifikacije i konfiguracijom implementacije.

## Preduvjeti

- **Docker** 24.0+ i **Docker Compose** v2.20+
- Računalo s najmanje **1 GB RAM-a** i **2 GB prostora na disku**
- Domensko ime (potrebno za OAuth povratne pozive i SSL)
- SMTP usluga za provjeru e-pošte (Resend je preporučen, ili bilo koji SMTP pružatelj)

## Brzi početak

### 1. Klonirajte repozitorij

```bash
git clone https://github.com/neurocanvas/neurocanvas.git
cd neurocanvas
```

### 2. Konfigurirajte okruženje

Stvorite `.env` datoteku u korijenu projekta:

```bash
# Required
AUTH_SECRET=your-secret-key-here
AUTH_ORIGIN=https://your-domain.com

# OAuth providers (at least one recommended)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret

# Email (for credentials auth and verification)
RESEND_API_KEY=re_your_api_key
EMAIL_FROM=NeuroCanvas <noreply@your-domain.com>
```

Pogledajte stranicu [Varijable okruženja](/docs/self-hosting/environment-variables) za potpunu referencu.

### 3. Pokrenite stek

```bash
docker compose up -d
```

Ovo pokreće tri servisa:
- **app** — NeuroCanvas aplikacija na portu 3000
- **db** — PostgreSQL 16 na portu 5432
- **redis** — Redis 7 na portu 6379

### 4. Provjera

```bash
curl http://localhost:3000/api/health
```

Očekivani odgovor:
```json
{
  "status": "ok",
  "timestamp": "2026-03-24T12:00:00.000Z",
  "version": "1.0.0"
}
```

## Konfiguracija Docker Composea

Priloženi `docker-compose.yml` definira potpuni stek:

```yaml
version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/neurocanvas
      - REDIS_URL=redis://redis:6379
      - AUTH_SECRET=${AUTH_SECRET}
      - AUTH_ORIGIN=${AUTH_ORIGIN:-http://localhost:3000}
      - GOOGLE_CLIENT_ID=${GOOGLE_CLIENT_ID}
      - GOOGLE_CLIENT_SECRET=${GOOGLE_CLIENT_SECRET}
      - GITHUB_CLIENT_ID=${GITHUB_CLIENT_ID}
      - GITHUB_CLIENT_SECRET=${GITHUB_CLIENT_SECRET}
      - RESEND_API_KEY=${RESEND_API_KEY}
      - EMAIL_FROM=${EMAIL_FROM:-NeuroCanvas <noreply@neurocanvas.app>}
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped

  db:
    image: postgres:16-alpine
    environment:
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=postgres
      - POSTGRES_DB=neurocanvas
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
```

### Ojačavanje za produkciju

::callout{type="danger"}
Zadana Docker Compose konfiguracija koristi zamjenske vjerodajnice. **Morate promijeniti sve zadane lozinke prije nego što aplikaciju izložite internetu.**
::

Za produkcijske implementacije, izmijenite compose datoteku:

1. **Promijenite zadanu lozinku baze podataka:**
   ```yaml
   db:
     environment:
       - POSTGRES_PASSWORD=a-strong-random-password
   ```
   Ažurirajte `DATABASE_URL` u servisu aplikacije da odgovara.

2. **Uklonite izložene portove** za db i redis (samo aplikacija treba vanjski pristup):
   ```yaml
   db:
     # Remove: ports: - "5432:5432"
   redis:
     # Remove: ports: - "6379:6379"
   ```

3. **Postavite lozinku za Redis:**
   ```yaml
   redis:
     command: redis-server --appendonly yes --requirepass your-redis-password
   ```
   Ažurirajte `REDIS_URL` na `redis://:your-redis-password@redis:6379`.

## Izgradnja iz izvornog koda

Dockerfile koristi višestupanjsku izgradnju:

1. **Faza izgradnje** — instalira ovisnosti, generira Prisma klijent, gradi Nuxt aplikaciju
2. **Produkcijska faza** — kopira samo izgrađeni izlaz i produkcijske ovisnosti

```bash
# Build the image
docker build -t neurocanvas:latest .

# Run standalone (requires external PostgreSQL and Redis)
docker run -d \
  --name neurocanvas \
  -p 3000:3000 \
  -e DATABASE_URL="postgresql://user:pass@host:5432/neurocanvas" \
  -e REDIS_URL="redis://host:6379" \
  -e AUTH_SECRET="your-secret" \
  -e AUTH_ORIGIN="https://your-domain.com" \
  neurocanvas:latest
```

Ulazna točka kontejnera automatski izvodi `prisma db push` radi primjene promjena sheme baze podataka prije pokretanja Node.js poslužitelja.

## Provjera ispravnosti

Aplikacija izlaže krajnju točku za provjeru ispravnosti na `GET /api/health`. Docker kontejner uključuje ugrađenu provjeru ispravnosti koja anketira tu krajnju točku svakih 30 sekundi:

```
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000/api/health || exit 1
```

Koristite ovu krajnju točku za provjere ispravnosti balansera opterećenja, nadzor dostupnosti i sonde spremnosti za orkestraciju kontejnera.

## Obrnuti proxy (nginx)

U produkciji postavite NeuroCanvas iza obrnutog proxyja za SSL prekid, HTTP/2 podršku i predmemoriranje.

### nginx konfiguracija

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # SSL hardening
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # Security headers
    add_header X-Frame-Options "DENY" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains" always;

    # Proxy to NeuroCanvas
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Timeouts for long-running AI operations
        proxy_read_timeout 120s;
        proxy_send_timeout 120s;
    }

    # Cache static assets
    location /_nuxt/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_cache_valid 200 7d;
        add_header Cache-Control "public, max-age=604800, immutable";
    }

    # Increase body size for map data uploads
    client_max_body_size 50M;
}
```

### SSL s Let's Encryptom

```bash
# Install certbot
sudo apt install certbot python3-certbot-nginx

# Obtain certificate
sudo certbot --nginx -d your-domain.com

# Auto-renewal is configured automatically
sudo certbot renew --dry-run
```

## Dodavanje nginxa u Docker Compose

Za uključivanje nginxa u isti compose stek:

```yaml
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
      - /etc/letsencrypt:/etc/letsencrypt:ro
    depends_on:
      - app
    restart: unless-stopped

  app:
    # Remove the ports mapping — nginx handles external traffic
    # ports:
    #   - "3000:3000"
    expose:
      - "3000"
```

## Ažuriranje

Za ažuriranje pokrenute implementacije:

```bash
# Pull latest changes
git pull origin main

# Rebuild and restart
docker compose up -d --build

# The entrypoint script runs database migrations automatically
```

Za ažuriranja bez prekida rada koristite obrazac plave-zelene implementacije ili orkestriranje kontejnera poput Docker Swarma ili Kubernetesa.
