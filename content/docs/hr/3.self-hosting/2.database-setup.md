# Postavljanje baze podataka

NeuroCanvas koristi **PostgreSQL** kao svoju primarnu bazu podataka i **Redis** kao neobavezni sloj predmemoriranja. Shemom baze podataka upravlja [Prisma ORM](https://www.prisma.io/).

## PostgreSQL zahtjevi

- **PostgreSQL 15+** (preporučuje se 16)
- Proširenje `pgcrypto` nije potrebno — generiranje ID-a koristi CUID-ove na razini aplikacije
- UTF-8 kodiranje (zadano)

## Stvaranje baze podataka

### Opcija 1: Docker (preporučeno)

`docker-compose.yml` priložen u repozitoriju automatski rukuje stvaranjem baze podataka:

```bash
docker compose up -d db
```

Ovo stvara PostgreSQL 16 instancu s:
- **Korisnik:** `postgres`
- **Lozinka:** `postgres` (promijenite u produkciji)
- **Baza podataka:** `neurocanvas`
- **Port:** 5432
- **Volumen podataka:** `postgres_data` (persists across container restarts)

### Opcija 2: Ručno postavljanje

Ako koristite postojeći PostgreSQL poslužitelj:

```sql
-- Connect as a superuser or a user with CREATEDB privileges
CREATE DATABASE neurocanvas;

-- Create a dedicated user (recommended)
CREATE USER neurocanvas_user WITH ENCRYPTED PASSWORD 'your-strong-password';
GRANT ALL PRIVILEGES ON DATABASE neurocanvas TO neurocanvas_user;

-- Connect to the new database and grant schema permissions
\c neurocanvas
GRANT ALL ON SCHEMA public TO neurocanvas_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO neurocanvas_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO neurocanvas_user;
```

Postavite `DATABASE_URL` u svom okruženju:
```
DATABASE_URL="postgresql://neurocanvas_user:your-strong-password@localhost:5432/neurocanvas"
```

### Opcija 3: Upravljani PostgreSQL

NeuroCanvas radi s upravljanim PostgreSQL pružateljima:

- **Neon** — bezsnaženički PostgreSQL s grananjem
- **Supabase** — PostgreSQL s ugrađenom autentifikacijom (koristite samo bazu podataka, ne Supabase Auth)
- **Railway** — jednoklik PostgreSQL provizioniranje
- **AWS RDS** / **Google Cloud SQL** / **Azure Database for PostgreSQL**

Koristite niz za povezivanje koji pruža vaš pružatelj hostinga kao `DATABASE_URL`. Osigurajte da je SSL omogućen za udaljene veze:

```
DATABASE_URL="postgresql://user:password@host:5432/neurocanvas?sslmode=require"
```

## Pokretanje migracija

Prisma upravlja shemom baze podataka. Primijenite shemu pri prvom postavljanju ili nakon ažuriranja:

### Docker implementacije

Skripta Docker ulazne točke automatski pokreće migracije pri pokretanju kontejnera:

```bash
# Runs automatically inside the container:
npx prisma db push --skip-generate --accept-data-loss
```

Za Docker implementacije nije potrebna ručna intervencija.

### Ručne implementacije

```bash
# Apply migrations to the database
npx prisma migrate deploy

# Alternative: push schema directly (for development or initial setup)
npx prisma db push
```

::callout{type="info"}
`prisma migrate deploy` primjenjuje datoteke migracija iz direktorija `prisma/migrations/` i preporučeni je pristup za produkciju. `prisma db push` izravno sinkronizira shemu bez stvaranja datoteka migracija i prikladniji je za razvoj.
::

### Generiranje Prisma klijenta

Nakon promjena sheme, ponovo generirajte TypeScript klijent:

```bash
npx prisma generate
```

Ovo se rješava automatski za vrijeme `npm install` (putem skripte `postinstall`) i za vrijeme Docker izgradnji.

## Pregled sheme

Shema baze podataka sastoji se od sljedećih tablica:

### Tablice autentifikacije

Ove tablice su neophodne za NextAuth.js (Auth.js) za upravljanje sesijama i OAuth:

| Tablica | Svrha |
|-------|---------|
| `User` | Korisnički računi. Sadrži `id` (CUID), `name`, `email`, `bio`, `emailVerified`, `image`, hashiranu `password` (za autentifikaciju s vjerodajnicama), `passwordChangedAt`, šifriran `totpSecret`, zastavicu `totpEnabled`, hashiran niz `backupCodes` i vremenske oznake. |
| `Account` | Povezani OAuth računi pružatelja. Jedan korisnik može imati više računa (npr. Google + GitHub). Kaskada pri brisanju korisnika. |
| `Session` | Aktivne sesije s tokenom i rokom trajanja. Kaskada pri brisanju korisnika. |
| `VerificationToken` | Tokeni za provjeru e-pošte i resetiranje lozinke. Jedinstven na `[identifier, token]`. |

### Tablice pohrane mapa

| Tablica | Svrha |
|-------|---------|
| `Map` | Mentalne mape. Koristi CUID generiran od klijenta kao primarni ključ. Pohranjuje podatke cijele mape kao JSON (`nodes`, `edges`, `camera`, `settings`, `rootNodeId`). Uključuje SHA-256 `checksum` za provjeru integriteta i `syncVersion` za optimistično upravljanje istovremenim pristupom. Podržava meko brisanje putem `deletedAt`. Indeksirano na `[userId, updatedAt]`. |
| `MapVersion` | Povijest verzija mapa. Svaka pohrana stvara novi unos verzije. Jedinstven na `[mapId, version]`. Kaskada pri brisanju mape. |
| `SyncState` | Metapodaci sinkronizacije po korisniku. Prati `lastSyncAt`, `lastDeviceId`, `totalMaps` i `totalByteSize`. Jedan-na-jedan s `User`. |

### Tablice dijeljenja

| Tablica | Svrha |
|-------|---------|
| `MapShare` | Tokeni djeljive veze za mape. Sadrži `token` (jedinstven), `mapId`, `userId`, neobavezno `expiresAt` i vremenske oznake. Omogućuje javni pristup samo za čitanje bez autentifikacije. |

### Tablice predložaka

| Tablica | Svrha |
|-------|---------|
| `Template` | Strukture mapa za ponovnu upotrebu. Sadrži `slug` (jedinstven), `title`, `description`, `category` (education/business/creative/planning/research), `tags`, `nodeCount`, `levelCount`, `nodes`/`edges`/`settings` (JSON), `previewData` (JSON), zastavicu `aiEnhanced`, zastavice `isSystem`/`isPublic`, `usageCount` i neobavezan `authorId`. Indeksirano na `[category]`. |

### Trezor vjerodajnica

| Tablica | Svrha |
|-------|---------|
| `Credential` | Šifrirani API ključevi za AI pružatelje (OpenAI, Anthropic, OpenRouter, prilagođeni). Ključevi su šifrirani na poslužiteljskoj strani i nikada se ne vraćaju u čistom tekstu. Jedinstven na `[userId, provider, label]`. |

### Dijagram veza između entiteta

```
User
 ├── Account[]         (1:N, CASCADE)
 ├── Session[]         (1:N, CASCADE)
 ├── Credential[]      (1:N, CASCADE)
 ├── Map[]             (1:N, CASCADE)
 ├── Template[]        (1:N, SET NULL on authorId)
 └── SyncState?        (1:1, CASCADE)

Map
 ├── MapVersion[]      (1:N, CASCADE)
 └── MapShare[]        (1:N, CASCADE)
```

Sve veze koriste `ON DELETE CASCADE` ako nije drugačije navedeno — brisanje korisnika uklanja sve njegove podatke. `authorId` predloška postavlja se na null pri brisanju korisnika kako bi sistemski predlošci ostali dostupni.

## Sigurnosna kopija i obnavljanje

### Automatizirane sigurnosne kopije s pg_dump

```bash
# Backup
pg_dump -h localhost -U postgres -d neurocanvas -F c -f backup_$(date +%Y%m%d_%H%M%S).dump

# Restore
pg_restore -h localhost -U postgres -d neurocanvas -c backup_20260324_120000.dump
```

### Docker sigurnosna kopija

```bash
# Backup from the running container
docker compose exec db pg_dump -U postgres neurocanvas > backup.sql

# Restore
docker compose exec -T db psql -U postgres neurocanvas < backup.sql
```

### Zakazane sigurnosne kopije

Koristite cron zadatak za automatizirane dnevne sigurnosne kopije:

```bash
# Add to crontab (crontab -e)
0 3 * * * pg_dump -h localhost -U postgres -d neurocanvas -F c -f /backups/neurocanvas_$(date +\%Y\%m\%d).dump 2>&1 | logger -t neurocanvas-backup
```

Zadržite najmanje 7 dnevnih sigurnosnih kopija i povremeno testirajte obnavljanje.

## Udruživanje veza

Za produkcijske implementacije s više instanci aplikacije ili bezsnaženičkim okruženjima koristite skupljač veza.

### PgBouncer

PgBouncer sjedi između vaše aplikacije i PostgreSQLa, ujedinjujući i ponovo koristeći veze baze podataka:

```bash
# Install PgBouncer
sudo apt install pgbouncer
```

Primjer `pgbouncer.ini`:
```ini
[databases]
neurocanvas = host=localhost port=5432 dbname=neurocanvas

[pgbouncer]
listen_addr = 127.0.0.1
listen_port = 6432
auth_type = md5
auth_file = /etc/pgbouncer/userlist.txt
pool_mode = transaction
max_client_conn = 200
default_pool_size = 20
```

Ažurirajte `DATABASE_URL` da usmjerava na PgBouncer:
```
DATABASE_URL="postgresql://postgres:postgres@localhost:6432/neurocanvas"
```

::callout{type="info"}
Koristite `pool_mode = transaction` (ne `session`) za kompatibilnost s Prismom. Prisma koristi named prepared statemente koji su ograničeni na sesije, ali transaction način to ispravno rješava s Prisminim zastavicom `pgbouncer=true`.
::

Za Prismu s PgBouncerjem dodajte parametar `pgbouncer` nizu za povezivanje:
```
DATABASE_URL="postgresql://postgres:postgres@localhost:6432/neurocanvas?pgbouncer=true"
```

### Ograničenja veza u Prismi

Prisma interno koristi skup veza. Konfigurirajte veličinu skupa putem niza za povezivanje:

```
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/neurocanvas?connection_limit=10"
```

Zadano je 10 veza. Za implementacije s jednom instancom, 10-20 je obično dovoljno. Smanjite ovo ako koristite PgBouncer.

## Postavljanje Redisa

Redis se koristi za predmemoriranje i neobavezan je ali se preporučuje za produkciju.

### Što Redis predmemorira

Redis se koristi za:

- **Vektore ugradnji** za izračun semantičke sličnosti
- **Predmemoriju AI odgovora** radi izbjegavanja redundantnih LLM API poziva
- **Stanje sinkronizacije** za brza čitanja bez opterećivanja PostgreSQLa
- **Distribuirana zaključavanja** radi sigurnosti pri istovremenim izmjenama mapa
- **Brojače ograničavanja brzine** za sprječavanje zlouporabe API-ja
- **Tokove događaja sinkronizacije** za obavijesti o promjenama u stvarnom vremenu
- **Metrike** za nadzor operacija sinkronizacije

### Docker Redis

Priloženi `docker-compose.yml` pokreće Redis 7 Alpine s trajnošću datoteke s dodavanjem:

```yaml
redis:
  image: redis:7-alpine
  command: redis-server --appendonly yes
  volumes:
    - redis_data:/data
```

Ovo osigurava trajnost između ponovnih pokretanja kontejnera. Za efemerno predmemoriranje gdje je gubitak podataka prihvatljiv, uklonite montiranje volumena.

### Upravljani Redis

NeuroCanvas radi s upravljanim Redis uslugama:

- **Upstash** — bezsnaženički Redis s cijenama po zahtjevu
- **Redis Cloud** — upravljani Redis od Redis Ltd.
- **AWS ElastiCache** / **Google Memorystore** / **Azure Cache for Redis**

Postavite varijablu okruženja `REDIS_URL` na niz za povezivanje koji pruža vaš pružatelj hostinga. Za TLS veze koristite protokol `rediss://`.

### Pokretanje bez Redisa

Ako `REDIS_URL` nije postavljen ili Redis nije dostupan, aplikacija elegantno vraća na alternativno ponašanje:

- Čitanja predmemorije vraćaju `null` (promašaj predmemorije)
- Pisanje u predmemoriju se tiho odbacuje
- Aplikacija nastavlja funkcionirati, ali s većom latencijom pri ponavljajućim operacijama
- Ograničavanje brzine sinkronizacije i distribuirana zaključavanja su onemogućeni
