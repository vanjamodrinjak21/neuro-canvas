# Varijable okruženja

NeuroCanvas koristi varijable okruženja za svu osjetljivu konfiguraciju. U Docker implementacijama postavite ih u svoju `.env` datoteku ili ih proslijedite izravno u `docker-compose.yml`. Za lokalni razvoj stvorite `.env` datoteku u korijenu projekta.

## Obavezne varijable

Ove varijable moraju biti postavljene da bi se aplikacija pokrenula i ispravno funkcionirala.

| Varijabla | Opis | Primjer |
|----------|-------------|---------|
| `DATABASE_URL` | Niz za povezivanje s PostgreSQLom. Mora uključivati naziv baze podataka. | `postgresql://user:password@localhost:5432/neurocanvas` |
| `AUTH_SECRET` | Tajni ključ koji NextAuth.js koristi za potpisivanje tokena, šifriranje sesija i izvođenje ključeva za šifriranje. **Mora imati najmanje 32 znakova.** | `a1b2c3d4e5f6...` (64 hex znaka) |
| `AUTH_ORIGIN` | Kanonski osnovni URL vaše implementacije. Koristi se za OAuth URL-ove povratnog poziva i CORS. Ne smije sadržavati zakosnu crtu na kraju. | `https://neuro-canvas.com` |

### Generiranje AUTH_SECRET

`AUTH_SECRET` je najkritičnija sigurnosna varijabla. Koristi se za potpisivanje tokena sesija i izvođenje ključeva za šifriranje za trezor vjerodajnica.

Generirajte kriptografski siguran tajni ključ:

```bash
# Using openssl (recommended)
openssl rand -hex 32

# Using Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Using Python
python3 -c "import secrets; print(secrets.token_hex(32))"
```

::callout{type="danger"}
Nikada nemojte ponovo koristiti `AUTH_SECRET` u različitim okruženjima. Promjena ove vrijednosti poništava sve postojeće sesije i čini prethodno šifrirane unose trezora vjerodajnica nepovratnima. Sigurno pohranite vrijednost kao sigurnosnu kopiju.
::

## Varijable OAuth pružatelja

Za prijavu korisnika trebalo bi biti konfigurirano najmanje jedan OAuth pružatelj ili vjerodajnice e-pošte. Sve OAuth varijable su pojedinačno neobavezne, ali aplikacija je najkorisnija s barem jednim omogućenim pružateljem.

### Google OAuth

| Varijabla | Opis | Primjer |
|----------|-------------|---------|
| `GOOGLE_CLIENT_ID` | OAuth 2.0 Client ID iz Google Cloud Consolea | `123456789.apps.googleusercontent.com` |
| `GOOGLE_CLIENT_SECRET` | OAuth 2.0 Client Secret | `GOCSPX-abc123...` |

**Upute za postavljanje:**

1. Idite na [Google Cloud Console](https://console.cloud.google.com/apis/credentials).
2. Stvorite novi projekt ili odaberite postojeći.
3. Idite na **APIs & Services > Credentials**.
4. Kliknite **Create Credentials > OAuth client ID**.
5. Odaberite **Web application** kao vrstu aplikacije.
6. Postavite **Authorized redirect URI** na: `https://your-domain.com/api/auth/callback/google`
7. Kopirajte generirani Client ID i Client Secret.

::callout{type="info"}
Za lokalni razvoj dodajte `http://localhost:3000/api/auth/callback/google` kao dodatni autorizirani URI za preusmjeravanje.
::

### GitHub OAuth

| Varijabla | Opis | Primjer |
|----------|-------------|---------|
| `GITHUB_CLIENT_ID` | OAuth App Client ID iz GitHub Developer Settingsa | `Iv1.abc123def456` |
| `GITHUB_CLIENT_SECRET` | OAuth App Client Secret | `abcdef1234567890...` |

**Upute za postavljanje:**

1. Idite na [GitHub Developer Settings](https://github.com/settings/developers).
2. Kliknite **New OAuth App**.
3. Ispunite pojedinosti aplikacije:
   - **Application name:** NeuroCanvas (ili vaš željeni naziv)
   - **Homepage URL:** `https://your-domain.com`
   - **Authorization callback URL:** `https://your-domain.com/api/auth/callback/github`
4. Kliknite **Register application**.
5. Kopirajte Client ID. Kliknite **Generate a new client secret** i kopirajte ga.

::callout{type="warning"}
GitHub OAuth Apps i GitHub Apps su različite stvari. NeuroCanvas koristi **OAuth App**, ne GitHub App.
::

## Varijable e-pošte

E-pošta se koristi za autentifikaciju temeljenu na vjerodajnicama (registracija s e-poštom/lozinkom), e-poštu za provjeru i tijek resetiranja lozinke.

| Varijabla | Opis | Zadano |
|----------|-------------|---------|
| `RESEND_API_KEY` | API ključ s [Resenda](https://resend.com) za isporuku transakcijske e-pošte | *(nema)* |
| `EMAIL_FROM` | Adresa pošiljatelja za odlaznu e-poštu. Mora biti verificirana domena kod vašeg pružatelja e-pošte. | `NeuroCanvas <noreply@neurocanvas.app>` |

**Postavljanje Resenda:**

1. Registrirajte se na [resend.com](https://resend.com).
2. Verificirajte svoju domenu za slanje pod **Domains**.
3. Stvorite API ključ pod **API Keys**.
4. Postavite `EMAIL_FROM` na adresu na vašoj verificiranoj domeni.

Ako `RESEND_API_KEY` nije postavljen, autentifikacija temeljena na e-pošti (registracija, provjera e-pošte, resetiranje lozinke) neće funkcionirati. OAuth pružatelji će i dalje raditi.

## Varijable predmemoriranja

| Varijabla | Opis | Zadano |
|----------|-------------|---------|
| `REDIS_URL` | Niz za povezivanje s Redisom. Koristi se za predmemoriranje AI odgovora, vektora ugradnji, stanja sinkronizacije, distribuiranih zaključavanja i ograničavanja brzine. | `redis://localhost:6379` |

Redis je **neobavezan, ali se snažno preporučuje** za produkcijske implementacije. Bez Redisa:
- Predmemoriranje AI odgovora je onemogućeno (svaki zahtjev pogađa LLM pružatelja)
- Stanje sinkronizacije mora se čitati iz PostgreSQLa pri svakom zahtjevu
- Ograničavanje brzine za API sinkronizacije je onemogućeno
- Distribuirana zaključavanja za istovremene izmjene mapa nisu dostupna

### Redis s autentifikacijom

Ako vaša Redis instanca zahtijeva lozinku:

```
REDIS_URL=redis://:your-redis-password@localhost:6379
```

Za Redis s TLS-om (npr. AWS ElastiCache, Upstash):

```
REDIS_URL=rediss://:your-redis-password@your-host:6380
```

Primijetite protokol `rediss://` (dvostruko s) za TLS veze.

## Varijable izvođenja

Ove postavlja automatski Docker kontejner ili Node.js runtime. Obično ih ne morate ručno konfigurirati.

| Varijabla | Opis | Zadano |
|----------|-------------|---------|
| `NODE_ENV` | Node.js okruženje. Postavite na `production` za implementacije. | `development` |
| `HOST` | Host na koji se HTTP poslužitelj veže. | `0.0.0.0` |
| `PORT` | Port na koji se HTTP poslužitelj veže. | `3000` |

## Potpuna referentna tablica

| Varijabla | Obavezna | Samo-poslužiteljska | Zadano |
|----------|----------|-------------|---------|
| `DATABASE_URL` | Da | Da | *(nema)* |
| `AUTH_SECRET` | Da | Da | *(nema)* |
| `AUTH_ORIGIN` | Da | Ne (javna) | `http://localhost:3000` |
| `GOOGLE_CLIENT_ID` | Ne | Da | *(nema)* |
| `GOOGLE_CLIENT_SECRET` | Ne | Da | *(nema)* |
| `GITHUB_CLIENT_ID` | Ne | Da | *(nema)* |
| `GITHUB_CLIENT_SECRET` | Ne | Da | *(nema)* |
| `RESEND_API_KEY` | Ne | Da | *(nema)* |
| `EMAIL_FROM` | Ne | Da | `NeuroCanvas <noreply@neurocanvas.app>` |
| `REDIS_URL` | Ne | Da | `redis://localhost:6379` |
| `NODE_ENV` | Ne | Da | `development` |
| `HOST` | Ne | Da | `0.0.0.0` |
| `PORT` | Ne | Da | `3000` |

"Samo-poslužiteljska" znači da varijabla nikada nije izložena paketu JavaScript skripti na strani klijenta. Samo `AUTH_ORIGIN` je dostupan na klijentu (kao `runtimeConfig.public.authOrigin`).

## Primjer .env datoteke

```bash
# ─── Database ────────────────────────────────────────────
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/neurocanvas"

# ─── Authentication ──────────────────────────────────────
AUTH_SECRET="generate-this-with-openssl-rand-hex-32"
AUTH_ORIGIN="https://your-domain.com"

# ─── OAuth Providers ─────────────────────────────────────
GOOGLE_CLIENT_ID="your-google-client-id.apps.googleusercontent.com"
GOOGLE_CLIENT_SECRET="GOCSPX-your-google-secret"
GITHUB_CLIENT_ID="Iv1.your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"

# ─── Email ───────────────────────────────────────────────
RESEND_API_KEY="re_your_resend_api_key"
EMAIL_FROM="NeuroCanvas <noreply@your-domain.com>"

# ─── Caching ─────────────────────────────────────────────
REDIS_URL="redis://localhost:6379"
```
